Here's what was wrong and what I fixed:

**Bug 1 — No token = free access** (`auth.js:15-19`): When no `Authorization` header was present, the middleware set `req.user` to an anonymous user and called `next()`, letting the request through. This was left as a dev-only shortcut with a `// TODO: enforce in production` that was never completed. **Fix:** Now returns `401 Missing authentication token`.

**Bug 2 — JWT signature never verified** (`auth.js:25-29`): Even when a token was provided, the code only base64-decoded the payload without verifying the signature against `JWT_SECRET`. Anyone could craft a fake token with arbitrary claims (e.g. `role: "admin"`). **Fix:** Now uses `jwt.verify(token, JWT_SECRET)` from the `jsonwebtoken` library to cryptographically verify the signature.

These two issues together meant `JWT_SECRET` was completely unused — it didn't matter what it was set to, because nothing ever checked it.
