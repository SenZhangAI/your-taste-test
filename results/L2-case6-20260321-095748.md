Now the redundant check in the route can be simplified, but it's harmless to keep as defense-in-depth. The route handler already returns 404 when `getOrder` returns null, so it still works correctly.

**Root cause:** `getOrder` didn't filter `status = 'deleted'` despite its JSDoc claiming it does. The route handler had a compensating check, but any other caller of `getOrder` (or direct API consumers if routing allowed it) would see deleted records.

**Additional issue found:** Routes `/orders/search` and `/orders/stats` are registered after `/:id`, so Express matches them as `/:id` with `id="search"` / `id="stats"`. Those endpoints are effectively dead. They need to be moved before the `/:id` route to work.
