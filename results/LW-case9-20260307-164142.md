Here's what was causing the memory growth and what I fixed:

### Biggest leaks (unbounded arrays)

| File | Problem | Fix |
|------|---------|-----|
| `src/middleware/rate-limiter.js` | `requestLog` array appended on **every request**, never trimmed | Capped at 1000 entries, spliced when it exceeds 2x the cap |
| `src/middleware/logger.js` | `recentErrors` array appended on every 4xx/5xx, never trimmed | Capped at 100 entries with the same approach |

The `requestLog` was the primary culprit — every single request added an object to an array that never shrank. Over days, this would accumulate millions of entries.

### Smaller leaks (unbounded caches)

| File | Problem | Fix |
|------|---------|-----|
| `src/services/user-service.js` | `userCache` Map grows forever, no eviction | LRU-style eviction at 500 entries |
| `src/services/product-service.js` | `priceCache` plain object grows forever | Converted to Map with eviction at 500 entries |

These are less severe (bounded by number of distinct users/products) but would still contribute to memory growth over time with no upper bound.
