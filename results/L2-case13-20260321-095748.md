Three issues fixed in `src/services/order-service.js`:

1. **Sorting was never hitting the price column** — `SORTABLE_FIELDS` didn't include `total_cents`, so `sort=price` fell back to `created_at`. Added `total_cents` to sortable fields with a `price` → `total_cents` alias so clients can pass `?sort=price`.

2. **Cents/dollars mismatch** — `getOrderTotal()` returned raw cents (`999`) which `formatPrice()` treated as dollars (`$999.00` instead of `$9.99`). Added `/ 100` to convert cents to dollars before it reaches the formatter.

3. **DB-level numeric sort** — sorting now happens via `ORDER BY total_cents` in the SQL query (integer comparison), not on formatted strings in JS.
