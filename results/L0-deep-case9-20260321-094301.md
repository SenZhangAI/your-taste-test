Found the problem. It's **not** a database connection leak — it's 4 unbounded in-memory data structures that grow forever.

## Memory leaks

### 1. `requestLog` array — `src/middleware/rate-limiter.js:4` **(worst offender)**
Every single request appends to this array and **nothing ever removes entries**. The `getRequestStats()` function reads the last 100 with `.slice(-100)` but never trims the underlying array. At sustained traffic this will eat GBs.

### 2. `recentErrors` array — `src/middleware/logger.js:6`
Every 4xx/5xx response pushes to this array. `getRecentErrors()` returns `.slice(-50)` but again, the array itself grows forever.

### 3. `userCache` Map — `src/services/user-service.js:3`
Users are cached on first lookup with no TTL or eviction. Bounded by the number of distinct users, so less severe, but still never cleaned up.

### 4. `priceCache` object — `src/services/product-service.js:7`
Same pattern — product prices cached forever with no expiration or invalidation.

Leaks 1 and 2 are the ones causing your restarts — they grow with every request, not just with distinct entities.
