Here's what changed:

**`order-service.js`** — `createOrder` now runs in a transaction that:
1. Looks up the product by `product_name` (with `forUpdate` row lock to prevent race conditions)
2. Validates the product exists and is active
3. Checks `stock >= quantity` — rejects with 409 if insufficient
4. Decrements `stock` and inserts the order atomically
5. Also sets `product_id` from the looked-up product, so you no longer need to pass it from the client

**`routes/orders.js`** — The POST handler catches stock/product errors and returns the appropriate status code (404 for unknown product, 409 for insufficient stock).
