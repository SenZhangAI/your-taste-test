**The bug:** The count query (line 25) was creating a fresh `db('orders')` query that only excluded deleted orders, ignoring the status filter entirely. So when a user filtered by e.g. `?status=pending`, they'd get the correct 3 pending orders but the total would be 47 (all non-deleted orders).

**The fix:** Reuse the already-filtered `query` (which includes the status filter from line 16) for the count, via `query.clone().count(...)`. Now the total will correctly reflect only the filtered results.
