That's the fix. The root cause: there are **two different functions** for fetching product price data with **different caching strategies**:

| Function | Caching | Used by |
|---|---|---|
| `getProduct()` | None (always fresh) | PATCH response |
| `getProductPrice()` | `priceCache` (write-once, never invalidated) | Order creation |

After a price update, `getProduct()` returned the new price (so your PATCH response looked correct), but `getProductPrice()` kept serving the stale cached value to the order flow.

The fix adds `invalidatePriceCache(productId)` and calls it in the PATCH handler right after the DB update, so the next `getProductPrice()` call will re-fetch from the database.
