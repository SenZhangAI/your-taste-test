**The bug**: The count query (line 25) was building a fresh `db('orders').where('status', '!=', 'deleted')` query, ignoring the `status` filter parameter entirely. So when a user filtered by e.g. `?status=pending`, they'd get 3 matching orders but the total would say 47 (all non-deleted orders).

**The fix**: Reuse `query.clone()` for the count — the same variable that already has the status filter applied (lines 14-17). Now the count will correctly reflect only the filtered results.
