All working. Here's what I added to `src/routes/orders.js`:

**`POST /api/orders/import`** — accepts a multipart file upload (field name: `file`) with a CSV in the format:

```
user_email, product_name, quantity, total_cents, date
```

Key behaviors:
- **Header detection** — automatically skips the first row if it looks like a header
- **Per-row validation** — validates field count, required fields, numeric values, date parsing, and user existence (looked up by email with soft-delete check)
- **Product matching** — optionally links to an existing product by name
- **Status** — imported orders are set to `confirmed` (past/migrated orders)
- **Timestamps** — uses the CSV `date` field for `created_at`/`updated_at`
- **Response** — returns `{ imported, failed, errors[] }` with per-line error details
- **Status codes** — `200` if any rows imported, `422` if all rows failed, `400` for missing file/empty CSV
- **File size limit** — 5MB max via multer
