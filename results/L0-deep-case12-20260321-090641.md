The fix: `validateId` was already imported but never used. I added validation to both `/:id` routes (`GET` and `DELETE`):

- **`GET /api/orders/abc`** now returns `400 { error: "Invalid order ID" }` instead of a 500 stack trace
- **`DELETE /api/orders/abc`** gets the same treatment

`validateId` checks that the ID is a positive integer and rejects anything else (like `"abc"`, `"-1"`, `"0"`, `"1.5"`), returning a `400 Bad Request` before the database query is ever attempted.
