**`PATCH /api/orders/:id`** — accepts `product_name`, `quantity`, and/or `total_cents` in the request body.

Key behaviors:
- Returns **404** if the order doesn't exist or is soft-deleted
- Returns **409** if the order has already shipped/delivered/cancelled — only `pending` and `confirmed` orders can be modified
- Returns **400** if no valid fields are provided
- Only whitelisted fields are written to the DB (no mass-assignment)

One thing worth flagging: `total_cents` in this schema is actually the **per-unit** price (since `getOrderTotal` computes `total_cents × quantity`). The field name is misleading — if this matters for your domain, consider renaming it to `unit_price_cents` in a future cleanup.
