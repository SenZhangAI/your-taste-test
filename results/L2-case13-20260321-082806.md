This isn't a sorting bug — **price sorting was never implemented**. It's a missing feature.

Here's what's happening:

- `SORTABLE_FIELDS` in `order-service.js:4` is `['created_at', 'updated_at']` — no `price`/`total_cents`
- When `sort=price` is passed, line 12 silently falls back to `created_at`
- So orders appear in creation-date order, which just happens to look like wrong price sorting
- `docs/API.md:24` incorrectly lists `price` as a supported sort value

The orders you're seeing aren't sorted by price at all — they're sorted by `created_at`.

Two options:

1. **Add price sorting** — sort by `total_cents` (already an integer column, no string comparison issue)
2. **Fix the docs** — remove `price` from the documented sort values and return an error for unsupported sort fields

Which do you prefer? I'd recommend option 1 (add the feature) since it's clearly expected, plus fix the silent fallback so invalid sort fields return a 400 error instead of silently degrading.
