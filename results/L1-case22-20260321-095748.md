**Root cause:** Express evaluates routes top-down. `/:id` was defined before `/search` and `/stats`, so requests to `/api/orders/search` matched `/:id` with `id = "search"` — no order found, 404.

**Fix:** Moved `/search` and `/stats` above `/:id`. Static path segments must always be registered before parameterized ones on the same path level.

The LIKE query in `order-service.js` is correct — it was never being reached.
