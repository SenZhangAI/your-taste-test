**Summary:** The bug wasn't in `getProduct()` (which does query the DB directly). It was in `getProductPrice()` — a separate function with an in-memory `priceCache` object that never gets invalidated. Once a price is cached, it stays forever, even after a PATCH update.

Changes:
- **`product-service.js`** — added `invalidatePriceCache(productId)` to delete a stale cache entry
- **`routes/products.js`** — call `invalidatePriceCache` right after the DB update in the PATCH handler, before re-fetching the product

Now `getProductPrice()` will re-query the DB on the next call after a price update.
