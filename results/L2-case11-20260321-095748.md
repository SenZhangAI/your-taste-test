Here's what changed:

**`order-service.js` — `createOrder`** now runs in a transaction that:
1. Looks up the product by `product_name` (with `forUpdate` row lock to prevent race conditions)
2. Validates the product exists and is active (404 if not)
3. Checks `product.stock >= quantity` (409 Conflict if insufficient)
4. Decrements stock and inserts the order atomically
5. Auto-sets `product_id` from the lookup — no longer needs it passed in

**`routes/orders.js`** — the POST handler now catches errors and returns the appropriate status code (`404` for unknown product, `409` for insufficient stock).
