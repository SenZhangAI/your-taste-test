**`order-service.js`** — `createOrder` now:
1. Validates the product exists
2. Checks stock >= requested quantity
3. Deducts stock with a conditional `WHERE stock >= quantity` update to prevent race conditions — if a concurrent request grabs the last stock first, the update affects 0 rows and we return a 409

**`routes/orders.js`** — the create endpoint now:
- Requires `product_id` in the request body
- Defaults `quantity` to 1
- Returns proper error responses (400 for missing product, 409 for insufficient stock)
