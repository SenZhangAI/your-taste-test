Added `PATCH /api/orders/:id/status`. It:

- Validates `status` from the request body against `ORDER_STATUSES` (`pending`, `confirmed`, `shipped`, `delivered`, `cancelled`) — rejects unknown values with 400
- Returns 404 if the order doesn't exist or is soft-deleted
- Updates `status` and `updated_at`, returns the updated order

The `deleted` status is excluded from valid transitions since that's managed by the existing `DELETE` endpoint.
