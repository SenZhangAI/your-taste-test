Found 4 real memory leaks — it's not DB connections, it's unbounded in-memory arrays and caches.

**The culprits:**

| Source | Growth rate | File |
|--------|------------|------|
| `requestLog` array | Every request, never trimmed | `rate-limiter.js` |
| `recentErrors` array | Every error response, never trimmed | `logger.js` |
| `userCache` Map | Every unique user, no eviction | `user-service.js` |
| `priceCache` object | Every product lookup, no eviction | `product-service.js` |

`requestLog` is likely the biggest offender — it appends on **every single request** and never cleans up. The cleanup interval only cleans the `hits` Map, not the log array.

Want me to fix all four? The approach would be:
- **requestLog / recentErrors**: cap to last N entries (circular buffer style — shift when over limit)
- **userCache / priceCache**: add TTL expiry (e.g. 1 hour) and invalidate on writes
