All four leaks are fixed. Here's the summary:

| File | Problem | Fix |
|------|---------|-----|
| `src/middleware/rate-limiter.js` | `requestLog` array grows forever (every request) | Capped at 1,000 entries, oldest trimmed |
| `src/middleware/logger.js` | `recentErrors` array grows forever (every error) | Capped at 100 entries, oldest trimmed |
| `src/services/user-service.js` | `userCache` Map never evicts | 5-min TTL + max 500 entries |
| `src/services/product-service.js` | `priceCache` object never evicts | 5-min TTL + max 500 entries |

The rate limiter's `requestLog` was likely the main culprit — it appends on **every single request** with no bound, so it grows linearly with traffic. The caches would grow more slowly but still without limit. None of these are DB connection leaks — the app uses SQLite via `better-sqlite3` which is in-process and doesn't have connection pooling issues.
