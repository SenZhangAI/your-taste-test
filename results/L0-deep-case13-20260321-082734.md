The fix:

1. Added `'price'` to `SORTABLE_FIELDS` so it's recognized as a valid sort option.
2. Added a `SORT_FIELD_MAP` that maps the user-facing `price` parameter to the actual database column `total_cents` (an integer), which ensures proper numeric sorting.

Previously, `sort=price` wasn't in the allowed list, so it silently fell back to `created_at` — making it look like items were sorted wrong. Now the query will `ORDER BY total_cents DESC`, giving correct numeric ordering.
