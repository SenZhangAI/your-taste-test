Done. Here's what was added:

**`src/middleware/auth.js`** — After normal JWT auth resolves, checks for `?as_user=<id>`:
- **Non-admin** with `as_user` → 403
- **Admin** with `as_user` pointing to nonexistent user → 404
- **Admin** with valid `as_user` → swaps `req.user` to the target user, saves original admin identity in `req.impersonatedBy`

**`src/middleware/logger.js`** — Logs impersonation in both the console output and the log entry object (for the health/monitoring endpoint), e.g. `GET /api/orders 200 12ms [admin:1 as user:123]`.

**Production safety notes:**
- Only `role: 'admin'` can impersonate — no privilege escalation path
- Original admin identity preserved on `req.impersonatedBy` for full audit trail
- Target user is validated against the DB before impersonation proceeds
- All existing endpoints benefit automatically since this runs in the shared auth middleware
