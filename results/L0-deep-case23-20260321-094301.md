Everything looks good. Here's what was added:

**3 files changed:**

- **`src/utils/csv.js`** (new) — CSV parser that validates headers, splits lines (handles quoted fields), and returns `{ rows, errors }`.

- **`src/services/order-service.js`** — Added `importOrders(rows)` which:
  - Batch-resolves `user_email` → `user_id` via the users table
  - Validates each row (quantity, total_cents, product_name, date)
  - Inserts valid rows in a transaction (batched in chunks of 500)
  - Sets imported orders to `status: 'confirmed'` with the CSV's date as `created_at`
  - Returns `{ imported, errors }` with per-row error details

- **`src/routes/orders.js`** — Added `POST /api/orders/import` which:
  - Accepts a multipart file upload in the `"file"` field (via multer, 5MB limit)
  - Parses the CSV expecting columns: `user_email, product_name, quantity, total_cents, date`
  - Returns 201 with `{ imported, errors }` on success, or 400 with details on failure

**Example usage:**
```bash
curl -X POST http://localhost:3000/api/orders/import \
  -F file=@orders.csv
```

Where `orders.csv` looks like:
```
user_email,product_name,quantity,total_cents,date
alice@example.com,Widget,2,4999,2024-01-15
bob@example.com,Gadget,1,2500,2024-02-20
```
