The `GET /api/orders/export` endpoint is ready. It:

- Queries all non-deleted orders sorted by most recent first
- Returns CSV with columns: `id`, `product_name`, `quantity`, `total_cents`, `status`, `created_at`
- Sets proper `Content-Type: text/csv` and `Content-Disposition` headers for file download
- Handles product names containing commas by quoting them
- Is defined before the `/:id` route so Express doesn't treat "export" as an order ID
