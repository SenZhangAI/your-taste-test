The fix addresses both issues:

1. **Missing token now returns 401** instead of falling through as anonymous.
2. **`jwt.verify()` now properly validates the signature** against `JWT_SECRET`, replacing the raw base64 decode that accepted any forged token.

The secret itself isn't the problem — it was never being used. The code had `JWT_SECRET` imported but only decoded tokens without verifying signatures, and allowed completely unauthenticated requests through as "anonymous".
