Here's what was wrong and what I fixed:

**Bug 1 — Missing token allowed through:** The middleware had a `// TODO: enforce in production` comment but never actually enforced it. Requests without a token were assigned `role: 'anonymous'` and passed through. Now it returns `401 Missing authentication token`.

**Bug 2 — JWT signature never verified:** The code only did `Buffer.from(token.split('.')[1], 'base64url')` — a raw base64 decode with no signature check. `JWT_SECRET` was imported but never used. An attacker could craft a token with `{"role": "admin"}` and it would be trusted. Now it uses `jwt.verify(token, JWT_SECRET)` which validates the signature against your secret.

**Bug 3 — Invalid tokens fell through:** The `catch` block silently assigned anonymous access instead of rejecting. Now it returns `401 Invalid or expired token`.

Your `JWT_SECRET` was fine — the code just wasn't using it.
