Here's what I changed:

**`order-service.js`** — `createOrder` now runs inside a transaction that:
1. Locks the product row with `forUpdate()` (prevents race conditions from concurrent orders)
2. Checks `product.stock >= quantity`
3. Deducts stock before inserting the order

**`routes/orders.js`** — The create endpoint now:
- Requires `product_id` in the request body
- Defaults `quantity` to 1
- Returns `409 Insufficient stock` or `404 Product not found` on failure

The `forUpdate()` row lock ensures two simultaneous orders can't oversell — the second transaction will wait for the first to commit before reading the updated stock.
