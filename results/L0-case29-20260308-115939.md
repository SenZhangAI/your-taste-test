Works. Here's what was added:

**`src/middleware/auth.js`** — After resolving the authenticated user from the JWT, the middleware now checks for a `?as_user=<userId>` query parameter:

1. **Admin-only gate**: If `as_user` is present but the caller's role isn't `admin`, returns `403 Forbidden`.
2. **User lookup**: Fetches the target user from the database via `getUser()`. Returns `404` if not found.
3. **Context swap**: Replaces `req.user` with the target user's info, and stores the original admin in `req.impersonatedBy` for audit/debugging.

**`src/middleware/logger.js`** — When impersonation is active, log lines include `[admin <id> impersonating user <id>]` for audit trail.

**Usage**: Any authenticated admin can append `?as_user=123` to any endpoint, e.g.:
```
GET /api/orders?as_user=123
POST /api/orders?as_user=456
```

The request will execute as if user 123/456 made it, and all downstream code using `req.user` works without changes.
