**Root cause:** Express evaluates routes in registration order. `/:id` is a wildcard that matches any single path segment — so `/search` and `/stats` were both caught by `/:id` before reaching their dedicated handlers.

**Fix:** Moved `/search` and `/stats` above `/:id`. Final route order:
1. `GET /` — list
2. `GET /search` — search by product name
3. `GET /stats` — admin stats
4. `GET /:id` — single order lookup
5. `POST /` — create
6. `DELETE /:id` — soft delete

The LIKE query in `searchOrders` is correct — it was never the problem.
