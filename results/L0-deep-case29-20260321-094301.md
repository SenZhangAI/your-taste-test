Done. The impersonation logic is in `src/middleware/auth.js:44-56`. Here's how it works:

- **Any endpoint** that goes through the auth middleware now supports `?as_user=123`
- **Admin-only**: Returns `403` if the requesting user's role isn't `admin`
- **Validation**: Returns `400` if the value isn't a positive integer
- **Identity swap**: Sets `req.user.id` to the target user ID and downgrades role to `user` so the admin sees exactly what that user would see
- **Audit trail**: The original admin identity is preserved on `req.adminUser` and as `req.user.impersonatedBy`
