The LIKE query is fine — the issue was route ordering. Express evaluates routes top-down, and `/:id` was matching before `/search` could be reached. The string `"search"` was being treated as an order ID, which obviously doesn't exist, hence the 404.

Moved `/search` above `/:id` so it gets matched first. The same issue applies to `/stats` (line 66) — it's also below `/:id` and would have the same problem if you tried to hit it.
