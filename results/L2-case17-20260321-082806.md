**`PATCH /api/orders/:id`** — updates an order's `product_name`, `quantity`, `total_cents`, or `product_id`.

Key guards:
- **404** if the order doesn't exist or is soft-deleted
- **409** if the order has already shipped/delivered/cancelled — only `pending` and `confirmed` orders can be modified
- **400** if no valid updatable fields are provided
- Whitelist approach — only the four allowed fields pass through, everything else (like `status`, `user_id`) is ignored
