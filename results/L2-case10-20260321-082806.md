**`PATCH /api/orders/:id`** — updates order status with transition validation.

**Status flow:**
```
pending → confirmed → shipped → delivered
   ↓          ↓
cancelled  cancelled
```

**Changes:**
- `src/utils/validators.js` — added `validateStatusTransition()` with a state machine defining allowed transitions
- `src/services/order-service.js` — added `updateOrderStatus(id, status)` 
- `src/routes/orders.js` — added `PATCH /:id` route that validates the transition before updating

**Request/Response example:**
```bash
PATCH /api/orders/1
{ "status": "confirmed" }

# 200 → returns updated order
# 400 → invalid transition (e.g. pending → shipped) or missing status
# 404 → order not found or deleted
```
