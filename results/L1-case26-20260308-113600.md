Removed the `priceCache` entirely. `getProductPrice()` now always queries the DB through `getProduct()`, so it's consistent and always returns the current price.

The cache comment said "avoid repeated DB lookups during batch operations" but there are no batch operations using it — and even if there were, a write-through cache or scoped batch cache would be the right pattern, not a module-level cache that never invalidates. If you ever need batch optimization, a better approach would be a short-lived cache scoped to the request/batch, not a persistent one.
