Three fixes:

**1. Stats query pushed to SQL** — instead of loading 500k rows into memory and aggregating in JS, `COUNT(*)` and `SUM()` now run in the database. Two lightweight queries instead of one massive data transfer. Should go from ~30s to milliseconds.

**2. Route ordering fixed** — `/stats` and `/search` are now defined *before* `/:id`. Previously Express matched `GET /orders/stats` as `/:id` with `id="stats"`, so the stats handler was unreachable. Same bug affected `/search`.

**3. Migration added** (`004-add-orders-status-index.js`) — index on `orders.status` to speed up the `WHERE status != 'deleted'` filter used by stats, list, and count queries.

Breadth-scan note: `listOrders` in `order-service.js:25` has a related issue — the count query ignores the `status` filter param (always counts all non-deleted), so pagination metadata is wrong when filtering by status. Separate bug, not blocking, but worth fixing if you're touching this area.
