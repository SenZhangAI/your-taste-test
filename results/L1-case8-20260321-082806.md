Fixed. Here's what was wrong and what changed:

**Root cause:** The auth middleware had two bugs that made it a complete no-op:
1. **Missing token → pass-through** (line 18-19): Requests without any token were allowed through as "anonymous" — a leftover dev shortcut with a `TODO: enforce in production` that was never addressed
2. **Token decoded but never verified** (line 25): Only did `Buffer.from(base64)` to read the JWT payload — never checked the signature against `JWT_SECRET`. Anyone could forge a token with any claims

**What changed:**
- No token → **401 rejected**
- Token present → **`jwt.verify()` checks the signature** against `JWT_SECRET`. Invalid/expired/forged tokens → 401
- Public paths (`/health`, `/api/products`) still skip auth as designed

**Breadth-scan note:** The routes (`orders.js`, `users.js`, `products.js`) don't check `req.user.role` for authorization — so once someone has a valid JWT, they can access everything. That's a separate concern (authZ vs authN), but worth adding role-based guards if you have admin-only endpoints.
