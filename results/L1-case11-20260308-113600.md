Here's what was added:

**`createOrder` now checks stock before accepting an order:**

1. **Looks up product by name** — returns 400 if product doesn't exist
2. **Checks `stock >= quantity`** — returns 409 (Conflict) with a clear message showing available vs requested
3. **Decrements stock atomically in a transaction** — the `WHERE stock >= orderQty` guard in the UPDATE prevents race conditions where two concurrent orders could oversell
4. **Auto-resolves `product_id`** from the product lookup (falls back to any explicitly passed value)

**`product-service.js`** — added `getProductByName(name)` for the lookup.

**Route handler** — catches the typed errors and returns the appropriate HTTP status.

One thing to note: the stock update uses `product.stock - orderQty` with a `WHERE stock >= orderQty` guard. For higher concurrency you'd want `stock = stock - ?` via `db.raw`, but for SQLite this is sufficient since writes are serialized.
