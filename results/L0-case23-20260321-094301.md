Both files look good. Here's what was added:

**`POST /api/orders/import`** — accepts a multipart form upload with field name `file`.

**Route** (`src/routes/orders.js:82-114`):
- Uses multer with memory storage (5MB limit)
- Validates the file is present, has a header row + data rows
- Validates CSV header contains the expected columns (`user_email, product_name, quantity, total_cents, date`)
- Parses rows by header position (supports columns in any order)

**Service** (`src/services/order-service.js:77-142`):
- Batch-resolves `user_email` → `user_id` via a single DB query
- Validates each row: required fields, positive integers, valid dates
- Inserts all valid rows in a transaction with status `confirmed`
- Preserves the original `date` as `created_at` (falls back to now)
- Returns `{ imported, failed, errors }` where errors include line numbers

**Example usage:**
```
curl -X POST http://localhost:3000/api/orders/import \
  -F file=@orders.csv
```

**Example response:**
```json
{
  "imported": 48,
  "failed": 2,
  "errors": [
    { "line": 5, "message": "user not found: unknown@example.com" },
    { "line": 12, "message": "quantity must be a positive integer" }
  ]
}
```
