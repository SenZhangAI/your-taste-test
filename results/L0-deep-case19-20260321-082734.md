Everything looks good. Here's a summary of all changes:

### Orders (bug fix + alignment)
- **`softDeleteOrder`** — now sets `deleted_at: db.fn.now()` in addition to `status: 'deleted'` (fixes the bug where `getUserOrders` filtered by `whereNull('deleted_at')` but it was never set)
- **`listOrders`**, **`searchOrders`**, stats query — switched from `where('status', '!=', 'deleted')` to `whereNull('deleted_at')`
- **Route handlers** (GET `/:id`, DELETE `/:id`) — switched from `order.status === 'deleted'` to `order.deleted_at`

### Products (new soft-delete support)
- **`listProducts`** — switched from `where('is_active', true)` to `whereNull('deleted_at')`
- **`softDeleteProduct`** — new function that sets `deleted_at`
- **Route handler** (GET `/:id`) — switched from `!product.is_active` to `product.deleted_at`
- **`src/db.js`** — added `deleted_at` column to products table in fallback schema
- **New migration `004-add-deleted-at-to-products.js`** — adds `deleted_at` column and migrates existing `is_active=false` rows

All three tables (users, orders, products) now consistently use `whereNull('deleted_at')` for soft-delete filtering.
