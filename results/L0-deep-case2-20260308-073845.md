Here's a summary of the changes:

**`src/services/order-service.js`:**
- `listOrders` — both query and count now use `.whereNull('deleted_at')` instead of `.where('status', '!=', 'deleted')`
- `getOrder` — added `.whereNull('deleted_at')` so it never returns soft-deleted orders (previously had no filter, despite the JSDoc claiming otherwise)
- `softDeleteOrder` — sets `deleted_at` timestamp instead of `status: 'deleted'`

**`src/routes/orders.js`:**
- Removed the now-redundant `order.status === 'deleted'` checks in the GET and DELETE handlers, since `getOrder` itself excludes deleted orders
