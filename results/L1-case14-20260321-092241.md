Here's what I found and fixed:

**Bug caught**: `getOrderTotal()` returns **cents** but `formatPrice()` expects **dollars**. Every GET endpoint was displaying `$2999.00` instead of `$29.99`. Simply copying this to POST would have propagated the bug.

**Changes made** (`src/routes/orders.js`):

1. **POST `/`** (line 49-53) — added `total` and `date` formatting to match GET response shape
2. **All endpoints** — added `/ 100` cents-to-dollars conversion before passing to `formatPrice`:
   - GET `/` (line 21)
   - GET `/:id` (line 37)
   - GET `/search` (line 64)
   - GET `/stats` revenue (line 74)
