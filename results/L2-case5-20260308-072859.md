**`GET /api/orders/export`** — streams all non-deleted orders as a CSV download.

Key decisions:
- **Streams from DB directly** instead of looping through the paginated API — simpler, no redundant count queries, and constant memory usage regardless of order count.
- **Placed before `/:id`** so Express doesn't treat "export" as an order ID.
- **Proper CSV escaping** — product names with commas or quotes are handled via double-quote wrapping.
- **Cents → dollars conversion** (`/ 100`) — the export correctly divides by 100. The existing `GET /` and `GET /:id` endpoints have a bug here: `getOrderTotal()` returns cents but `formatPrice()` treats the value as dollars, so prices display 100x too high. Worth fixing separately.
