The endpoint is implemented. Here's what was added:

**`POST /api/orders/import`** — accepts a multipart file upload (field name: `file`) containing a CSV.

**CSV format:**
```
user_email, product_name, quantity, total_cents, date
alice@example.com, Widget Pro, 2, 4999, 2024-01-15
```

**Behavior:**
- Header row is auto-detected and skipped (if it contains `user_email`)
- Each row is validated: all 5 columns required, quantity/total_cents must be positive integers, date must be parseable
- Users are looked up by email — rows with unknown emails are reported as errors
- Products are matched by name to set `product_id` (optional — orders still import if product isn't found)
- Imported orders get status `confirmed` (past orders, not pending)
- `created_at` is set from the CSV `date` column to preserve the original order date
- 5MB file size limit

**Response:**
```json
{ "created": 3, "errors": [{ "row": 4, "message": "User not found: bad@email.com" }] }
```

Returns 200 on success (even partial), 422 if all rows failed, 400 for missing/empty file.
