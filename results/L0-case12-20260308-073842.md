The fix adds ID validation to both the GET `/:id` and DELETE `/:id` routes using the already-imported `validateId` helper. Now `GET /api/orders/abc` will return a `400` with `{"error": "Invalid order ID"}` instead of a 500 with a stack trace.

`validateId` (in `src/utils/validators.js:5`) parses the ID as an integer and returns `null` if it's not a positive integer, so non-numeric, negative, or zero IDs are all rejected before hitting the database.
