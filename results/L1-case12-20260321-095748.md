Fixed. `GET /api/orders/abc` will now return `400 { "error": "Invalid order ID" }` instead of a 500.

The root cause: `validateId` was imported in orders.js but never used — the raw `req.params.id` string was passed directly to the DB query, which throws on non-numeric input.

**Breadth-scan findings** — the same missing validation existed on every `/:id` route in the codebase, so I fixed all of them:

- **orders.js**: `GET /:id`, `DELETE /:id`
- **users.js**: `GET /:id`
- **products.js**: `GET /:id`, `PATCH /:id`
