Fixed both POST `/` and GET `/:id`. All three order endpoints now return the same clean shape:

```js
{ id, product, quantity, total: "$29.99", status, date: "2026-03-01" }
```

GET `/:id` previously had the same issue — it spread `...order` which leaked raw DB fields (`total_cents`, `created_at`, `product_name`, etc.) alongside the formatted ones.

One thing worth noting: `getOrderTotal` JSDoc says it returns **cents**, but `formatPrice` expects **dollars**. If the field `total_cents` truly stores cents, then GET has been displaying `$2999.00` not `$29.99`. Either the field name is wrong (actually stores dollars) or there's a missing `/100` conversion. Worth verifying against actual data.
