Here's what was going on and what I changed:

**Root cause:** Price sorting was never implemented. The `SORTABLE_FIELDS` array only had `created_at` and `updated_at`, so `sort=price` silently fell back to `created_at`. The ordering you saw wasn't string-vs-numeric — it was just creation order.

**Fix in `src/services/order-service.js`:**
- Added `total_cents` to `SORTABLE_FIELDS` so sorting by the numeric cents column works correctly
- Added a `SORT_ALIASES` map so `sort=price` resolves to the `total_cents` database column, which is an integer and sorts numerically
