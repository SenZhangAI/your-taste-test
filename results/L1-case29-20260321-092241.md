Done. Here's what changed in `src/middleware/auth.js`:

**How it works:**
- After normal JWT auth, checks for `?as_user=<id>` query param
- **Non-admin** requests with `as_user` → `403 Forbidden`
- **Admin** with invalid user ID → `404 Not Found`
- **Admin** with valid user ID → `req.user` becomes the target user, original admin saved in `req.impersonator` (useful for audit logging)

**Breadth-scan:** The middleware is `async` now (was sync). Express handles async middleware fine — errors will propagate. No other callers reference `authenticate` besides `index.js` where it's used as `app.use(authenticate)`, so no breaking change. The `as_user` param won't conflict with any existing query params on routes (checked: orders uses `page`, `limit`, `sort`, `status`, `q`; products/users don't overlap).
