Here's what was added:

**Service layer** (`src/services/order-service.js`): New `updateOrder(id, fields)` function that updates the given fields and sets `updated_at`.

**Route** (`src/routes/orders.js`): `PATCH /api/orders/:id` that:
- Returns **404** if the order doesn't exist or is soft-deleted
- Returns **409** if the order status is `shipped` or `delivered` (can't modify after shipment)
- Returns **400** if no valid fields (`product_name`, `quantity`, `total_cents`) are provided
- Only includes fields that were actually sent in the request body, so partial updates work
