The endpoint is ready. Key design decisions:

- **Direct DB query** — bypasses the paginated `listOrders()` to avoid looping through ~100 pages for 5000+ rows
- **Streaming** — uses `knex.stream()` so rows are piped to the response as they're read, keeping memory usage constant regardless of dataset size
- **Route placement** — `/export` is registered before `/:id` so Express doesn't treat "export" as an order ID
- **CSV escaping** — handles commas, quotes, and newlines in field values
