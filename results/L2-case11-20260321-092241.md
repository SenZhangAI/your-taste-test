Here's what the stock check does:

1. **Looks up the product by `product_name`** — returns 400 if not found
2. **Checks `product.stock >= quantity`** — returns 409 if insufficient
3. **Decrements stock with a conditional `WHERE stock >= qty`** — prevents race conditions where two concurrent orders could both pass the initial check but oversell. If the atomic update affects 0 rows, it returns 409
4. **Auto-fills `product_id`** from the looked-up product if not provided in the request
