**Root cause**: `priceCache` in `product-service.js` is a write-once, never-invalidated in-memory cache. `getProductPrice()` populates it on first call, and the `PATCH` endpoint updates the DB but never clears the cached entry. So all subsequent price lookups return the stale value.

**Fix**: Added `invalidatePriceCache(productId)` and call it in the price update route right after the DB write, so the next `getProductPrice()` call fetches the fresh value from the DB.
