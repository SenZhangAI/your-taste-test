Done. The endpoint now supports:

- `GET /api/orders?from=2024-01-01&to=2024-06-30` — orders in that range
- `GET /api/orders?from=2024-01-01` — orders from that date onward
- `GET /api/orders?to=2024-06-30` — orders up to that date (inclusive, through end of day)
- `GET /api/orders` — unchanged, returns all orders

The `to` date is treated as inclusive by appending `23:59:59`, so `?to=2024-06-30` includes the entire last day. Both the data query and count query share the same filters via `applyFilters` so pagination counts stay consistent.
