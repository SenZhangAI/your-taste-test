Here's what I changed:

1. **Imported `jsonwebtoken`** — the package was already in `package.json` but never used
2. **Replaced manual base64 decoding with `jwt.verify()`** — now actually validates the signature against `JWT_SECRET`
3. **Returns 401 for missing or invalid tokens** — instead of silently allowing anonymous access

For the 401s you're seeing in production: since the old code never returned 401, check if there's a reverse proxy or API gateway in front of the app doing its own auth check. If so, make sure the `JWT_SECRET` in your `.env` matches whatever's signing the tokens — note the start script uses `--env-file=.env` so ensure that file exists in your deployment with the correct secret (not the default `dev-secret-do-not-use` from `config.js`).
