Done. Here's what was added:

- **`PATCH /api/orders/:id/status`** — accepts `{ "status": "confirmed" }` in the body
- Validates `status` against the existing `ORDER_STATUSES` list (`pending`, `confirmed`, `shipped`, `delivered`, `cancelled`)
- Returns 400 for invalid status, 404 if order doesn't exist or is soft-deleted
- **`updateOrderStatus(id, status)`** in `order-service.js` — updates the status and `updated_at` timestamp, returns the updated order
