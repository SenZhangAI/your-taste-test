Here's the issue: `product-service.js` has a `priceCache` object (line 7) that caches product prices forever with no invalidation. Your `PATCH /api/products/:id` route updates the DB directly but never clears this cache, so `getProductPrice()` keeps returning the old cached value.

**The fix** (2 changes):

1. **`src/services/product-service.js`** — added `clearPriceCache(productId)` that deletes the stale entry
2. **`src/routes/products.js`** — call `clearPriceCache(req.params.id)` right after the DB update, before returning the response

Now when a price is updated, the next call to `getProductPrice()` will miss the cache and fetch the fresh value from the DB.
