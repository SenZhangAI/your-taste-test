## Summary

Not a DB connection leak. Three issues:

| Source | Problem | Fix |
|--------|---------|-----|
| `rate-limiter.js` `requestLog` | Every request appended, never trimmed — grows forever | Trim to last 1000 entries every 2 min (alongside existing cleanup interval) |
| `logger.js` `recentErrors` | Every error appended, never trimmed | Cap at 50 entries (trim when exceeds 100) |
| `orders.js` `/stats` | Loads ALL order rows into JS to count/sum | Replaced with SQL aggregate queries (`COUNT`, `SUM`, `GROUP BY`) |

The `requestLog` is almost certainly your main culprit — it accumulates an object for every single request with no upper bound. Over days of traffic that's millions of entries sitting in heap.
