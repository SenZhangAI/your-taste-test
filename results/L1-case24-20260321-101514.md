Done. Here's what was added:

**Schema** (`db.js`): `refund_cents` (nullable int) and `cancelled_at` (nullable timestamp) columns on orders.

**Service** (`order-service.js`): `cancelOrder(id, refundCents)` — sets status to `cancelled`, records refund amount and timestamp.

**Endpoint** (`routes/orders.js`):
- `POST /api/orders/:id/cancel` — accepts optional `refund_cents` in body (defaults to full refund if omitted). Validates refund is between 0 and order total. Returns the cancelled order with formatted `refund` field.
- `GET /api/orders/:id` — now includes `refund` and `cancelled_at` for cancelled orders.

**Usage examples:**
```bash
# Full refund (default)
POST /api/orders/42/cancel

# Partial refund
POST /api/orders/42/cancel
{ "refund_cents": 1500 }
```

Note: Since the schema is created fresh via `initDB()`, the new columns will be there on next DB init. If you have an existing database, you'll need to run a migration to add the columns.
