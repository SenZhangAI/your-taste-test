The impersonation feature is implemented in `src/middleware/auth.js`. Here's what it does:

- **`?as_user=123`** on any endpoint triggers impersonation
- **Admin-only**: Returns 403 if the caller's JWT `role` is not `admin`
- **Validates** the target user ID (must be numeric, must exist in DB)
- **Swaps `req.user`** to the target user's identity so all downstream handlers act as that user
- **Preserves audit trail**: The original admin identity is saved in `req.adminUser` and `req.user.impersonatedBy`
