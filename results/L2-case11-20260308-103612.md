Here's what changed:

**`order-service.js` `createOrder`** ‚Äî before inserting, it now:
1. Looks up the product by `product_name` (active products only)
2. Returns 404 if product not found, 409 if insufficient stock
3. Decrements stock and inserts the order in a **single transaction** with a `WHERE stock >= qty` guard to prevent race conditions
4. Auto-populates `product_id` from the lookup, so callers don't need to pass it

**`routes/orders.js`** ‚Äî catches service errors with a `status` property and returns them as proper HTTP responses.
