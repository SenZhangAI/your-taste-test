### What changed:

1. **`jwt.verify()` replaces manual base64 decode** — signature is now validated against `JWT_SECRET`
2. **No token → 401**, not anonymous passthrough
3. **Invalid/expired token → 401**, not silent fallback to anonymous
4. **Public paths restricted to GET only** — prevents unauthenticated POST/DELETE to `/api/products` if product write endpoints are added later

### Breadth-scan — additional risk:

**`config.js:3`** has `JWT_SECRET = process.env.JWT_SECRET || 'dev-secret-do-not-use'`. If the env var is somehow unset in prod, it silently falls back to a hardcoded secret. This is a secondary concern — you said the env var is set — but worth flagging. If you want, I can make it throw on startup when `JWT_SECRET` is missing instead of falling back.
