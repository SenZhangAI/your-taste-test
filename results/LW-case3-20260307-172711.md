Done. `RATE_LIMIT` now defaults to **500** and reads from `process.env.RATE_LIMIT`, consistent with how `PORT` and `JWT_SECRET` are already handled.

A couple things worth noting:

1. **`.env.example` says `RATE_LIMIT=10`** — you may want to update that to `500` to match the new default, or whatever value makes sense for your environments.

2. **`requestLog` in `rate-limiter.js:4`** is an unbounded array that grows with every request and is never trimmed. In production this will leak memory over time. The `hits` Map is fine (the `setInterval` cleanup handles it), but `requestLog` has no such cleanup. Worth capping or replacing with a counter if you don't need the full request history.
