Everything looks good. Here's what was added:

**`PATCH /api/orders/:id`** — updates an order before it ships. Two files changed:

**`src/services/order-service.js`** — new `updateOrder(id, updates)` function that:
- Returns 404 if the order doesn't exist or is deleted
- Returns 409 if the order has already shipped/delivered/cancelled (only `pending` and `confirmed` orders can be modified)
- Whitelists updatable fields (`product_name`, `quantity`, `total_cents`, `product_id`) to prevent injecting arbitrary columns
- Returns 400 if no valid fields are provided
- Sets `updated_at` on every update

**`src/routes/orders.js`** — new PATCH route that:
- Validates the `:id` param with `validateId`
- Validates `quantity` and `total_cents` are positive integers (if provided)
- Validates `product_name` is a string (if provided)
- Returns the formatted order on success (same shape as `GET /:id`)
