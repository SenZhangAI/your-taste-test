Here's what I changed:

**`src/services/order-service.js`** — `createOrder` now runs inside a transaction that:
1. Looks up the product by `product_id`
2. Returns 400 if the product doesn't exist
3. Returns 409 if `product.stock < quantity`
4. Deducts stock and inserts the order atomically

**`src/routes/orders.js`** — The POST route now:
- Requires `product_id` in the request body
- Defaults `quantity` to 1 if not provided
- Catches stock/product errors and returns the appropriate status code

The transaction ensures no overselling under concurrent requests — if two orders race, one will see the already-deducted stock and fail.
