**Not a caching issue.** The bug is that `getOrder` in `order-service.js:32` wasn't filtering out soft-deleted records despite the JSDoc claiming it did. The query was just `where({ id })` with no status check.

The route handler had a compensating `order.status === 'deleted'` guard, which is why the `GET /orders/:id` endpoint *appeared* to work — but any other code calling `getOrder` (like `createOrder` which calls it to return the newly created order) would have returned deleted records.

**Fix:** Added `.whereNot('status', 'deleted')` to `getOrder` and removed the now-redundant status checks from the route handlers.
