**The bug:** Line 25 was building a fresh `db('orders')` query for the count instead of reusing the `query` variable that already has the status filter applied. So the count always returned the total of all non-deleted orders regardless of the active filter.

**The fix:** Changed the count query to use `query.clone()` so it inherits the same status filter applied on lines 14-16. Now when a user filters by e.g. "pending", the total will correctly reflect only pending orders.
