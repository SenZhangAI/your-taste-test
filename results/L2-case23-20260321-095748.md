**`POST /api/orders/import`** — accepts a CSV file upload (multipart form field `file`).

**Route** (`src/routes/orders.js:46-77`):
- Uses multer with in-memory storage, 5MB limit
- Validates file presence, header row with required columns (`user_email, product_name, quantity, total_cents, date`)
- Parses CSV and delegates to service
- Returns `{ imported, failed, errors }` with per-row error details (line numbers)

**Service** (`src/services/order-service.js:77-142`):
- Batch-resolves `user_email` → `user_id` via single DB query
- Validates each row: required fields, positive quantity, non-negative total_cents, valid date
- Inserts all valid rows in a single transaction with `status: 'confirmed'` (these are past/migrated orders)
- `created_at` set from CSV `date` column to preserve original order dates

**Usage:**
```bash
curl -X POST /api/orders/import \
  -F file=@orders.csv
```
