Done. Here's what changed:

**`src/services/order-service.js`** — `createOrder` now:
1. Validates the product exists via `product_id`
2. Checks stock before attempting the transaction (fast fail)
3. Uses a DB transaction with an atomic `WHERE stock >= quantity` guard to deduct stock and insert the order — prevents race conditions where two concurrent orders oversell

**`src/routes/orders.js`** — Order creation now:
1. Requires `product_id` in the request body
2. Defaults `quantity` to 1
3. Returns `409 Conflict` for insufficient stock, `400` for missing product

The double-check pattern (optimistic read + pessimistic `WHERE` in the transaction) avoids unnecessary transaction overhead for obviously out-of-stock products while still being race-safe.
