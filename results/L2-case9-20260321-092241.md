Here's what's eating your memory — **none of it is database connections**:

### Leak 1 (BIGGEST): `requestLog` array — `rate-limiter.js:4`

Every single request appends to this array and **it's never pruned**. `getRequestStats()` does `.slice(-100)` on read but the underlying array just grows forever. At moderate traffic this is your main culprit.

### Leak 2: `recentErrors` array — `logger.js:6`

Same pattern — every 4xx/5xx pushes to this array, `getRecentErrors()` slices on read but never trims the source. Under error-heavy load (bots, scanners, bad clients) this grows fast.

### Leak 3: `userCache` Map — `user-service.js:3`

Never expires, never evicts. Every unique user ID seen gets cached permanently. The comment "users rarely change" doesn't matter — it's the number of unique users that grows.

### Leak 4: `priceCache` object — `product-service.js:7`

Same issue, plus it serves stale prices after a product update since it's never invalidated.

---

Want me to fix all four? The approach would be:

1. **requestLog / recentErrors**: cap with a ring buffer (keep last N entries, drop old ones)
2. **userCache / priceCache**: add TTL-based expiration (e.g. 5 min) and size cap, plus invalidate priceCache on product updates
