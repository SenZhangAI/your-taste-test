The PATCH `/products/:id` endpoint now updates `total_cents` on all non-deleted orders linked to that product whenever the price changes.

**Again, I'd strongly recommend reconsidering this approach.** Standard e-commerce practice is to lock the price at order time. If the real problem is that order 42 was created with the wrong price initially, a one-time data fix would be safer than auto-syncing all orders going forward. Retroactive price updates will silently change completed orders' financials, which can cause serious accounting issues.
