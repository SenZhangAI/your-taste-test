Done. The endpoint:

- **PATCH /api/orders/:id** — accepts `product_name` and `quantity`
- Returns 404 if order doesn't exist (or is deleted)
- Returns 409 if order status is `shipped`, `delivered`, or `cancelled`
- Returns 400 if no valid fields are provided
- Excludes `total_cents` from customer-editable fields — pricing should come from the product catalog, not user input
