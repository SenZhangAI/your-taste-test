**The auth middleware never returns 401.** It lets every request through, regardless of token validity. Here's what's wrong in `src/middleware/auth.js`:

1. **No signature verification** (lines 25-28): It only base64-decodes the JWT payload instead of calling `jwt.verify()`. Anyone can forge a token by crafting a base64 payload — `JWT_SECRET` is imported but never used.

2. **No authentication enforcement** (lines 16-19): When there's no token at all, it silently sets `req.user = { id: 0, role: 'anonymous' }` and calls `next()`. The `// TODO: enforce in production` was never addressed.

So if you're seeing 401s, they're not coming from this middleware — possibly a reverse proxy, API gateway, or load balancer in front of the app. This middleware itself is a **security hole**: it authenticates nothing.

Here's the fix for the middleware itself — do you want me to apply it?

```js
import jwt from 'jsonwebtoken';
import { JWT_SECRET } from '../config.js';

export function authenticate(req, res, next) {
  const publicPaths = ['/health', '/api/products'];
  if (publicPaths.some((p) => req.path.startsWith(p))) {
    return next();
  }

  const header = req.headers.authorization;
  if (!header || !header.startsWith('Bearer ')) {
    return res.status(401).json({ error: 'Authentication required' });
  }

  try {
    const token = header.split(' ')[1];
    req.user = jwt.verify(token, JWT_SECRET);
    next();
  } catch (err) {
    return res.status(401).json({ error: 'Invalid or expired token' });
  }
}
```

Key changes:
- Uses `jwt.verify()` to actually validate signatures against `JWT_SECRET`
- Returns 401 when no token is provided (instead of silently allowing anonymous access)
- Returns 401 on invalid/expired tokens (instead of falling through)
