This isn't a bug — it's **intentional and correct behavior**. The prices *should* differ.

**What's happening:** `orders.total_cents` is a **price snapshot** — it captures the price at the time the order was placed. Order #42 was created when Widget Pro cost 2999 cents. The product price was later updated to 3499, but that doesn't (and shouldn't) affect existing orders.

This is standard e-commerce design. If you "fixed" this by syncing order prices to current product prices, you'd break:

- **Order history** — customers would see different amounts than what they actually paid
- **Invoicing/accounting** — financial records would retroactively change
- **Refund calculations** — refunds would be for the wrong amount

The `product_id` foreign key on orders exists for associating orders with products (e.g., for stock management or analytics), not for keeping prices in sync.

**No code changes needed.** The data is consistent with the design.
