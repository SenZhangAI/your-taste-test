**`POST /api/orders/import`** — accepts a multipart form upload with a `file` field containing a CSV.

**Key details:**

- **CSV format**: `user_email, product_name, quantity, total_cents, date` — header row is auto-detected and skipped
- **User lookup**: Resolves `user_email` to `user_id` via a single pre-fetched query against the users table (case-insensitive)
- **Validation**: Each row is validated (required fields, positive integers, valid date). Invalid rows are skipped with per-row error reporting
- **Atomicity**: All valid rows are inserted in a single database transaction via `bulkCreateOrders`
- **Status**: Imported orders get `status: 'completed'` since they're historical
- **Response**: Returns `{ imported, skipped, errors? }` with 201 on success
- **Limits**: 5MB max file size via multer
- **Route placement**: Registered before `/:id` to avoid the param route capturing `/import`
