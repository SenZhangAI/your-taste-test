Fixed. The changes:

1. **Use `jsonwebtoken` for proper verification** — `jwt.verify()` checks the signature against `JWT_SECRET` instead of just base64-decoding the payload
2. **Reject unauthenticated requests** — returns 401 when token is missing or invalid, instead of silently passing through as anonymous

If your production `JWT_SECRET` env var isn't set, it falls back to `'dev-secret-do-not-use'` (in `config.js:3`), which would reject any token signed with the real secret. Make sure `JWT_SECRET` is set in your production environment.
