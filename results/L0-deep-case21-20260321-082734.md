The stale `userCache` was serving a pre-deletion snapshot of the user object, so `deleted_at` was always `null` for cached users. The route's guard (`user.deleted_at`) never triggered, and the full profile + orders were returned.

The orders endpoint itself wasn't the leak — it was `GET /api/users/:id` returning orders alongside the stale cached user. The orders query in `getUserOrders` actually filters by `deleted_at` correctly; the problem was entirely that the user object was served from a never-invalidated cache.
