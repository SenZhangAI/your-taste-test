**Root cause:** Express route ordering. `GET /:id` was registered before `GET /search` and `GET /stats`, so requests to `/api/orders/search` matched `/:id` with `id = "search"` — no order found, 404.

**Fix:** Moved `/search` and `/stats` above `/:id`. Static path segments must always come before parameterized routes in Express. The LIKE query itself was correct.
