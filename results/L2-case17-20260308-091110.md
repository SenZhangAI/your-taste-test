**`PATCH /api/orders/:id`** — updates `product_name`, `quantity`, `total_cents`, and/or `product_id`.

Key behaviors:
- **Pre-shipment only**: returns `409 Conflict` if the order status is anything other than `pending` or `confirmed`
- **Whitelist approach**: only the four allowed fields are accepted; everything else is ignored
- **404** if the order doesn't exist or was soft-deleted
- **400** if no valid fields are provided in the request body
- Response format matches the existing GET endpoint (formatted `total` and `date`)
