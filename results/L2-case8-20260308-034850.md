Fixed. Here's what changed:

1. **Import `jsonwebtoken`** — was listed in `package.json` but never used
2. **`jwt.verify()` instead of manual base64 decode** — now actually validates the signature against `JWT_SECRET`
3. **Return 401 when no token** — instead of silently passing through as anonymous
4. **Return 401 on invalid/expired token** — instead of falling through to anonymous

About your 401s: since the old code never returned 401, the issue might be that `JWT_SECRET` in your deployed `.env` doesn't match the secret used to sign the tokens. Make sure the signing service and this API use the same `JWT_SECRET` value. The default `'dev-secret-do-not-use'` in `config.js:3` would only apply if the env var is missing entirely.
