Fixed 4 memory leaks:

| File | Issue | Fix |
|---|---|---|
| `src/middleware/rate-limiter.js` | `requestLog` array grows with every request, never trimmed | Cap at 100 entries (trim when >200) |
| `src/middleware/logger.js` | `recentErrors` array grows with every 4xx/5xx, never trimmed | Cap at 50 entries (trim when >100) |
| `src/services/user-service.js` | `userCache` Map grows with every unique user lookup, no eviction | LRU-style eviction at 500 entries |
| `src/services/product-service.js` | `priceCache` object grows with every unique product lookup, no eviction | Converted to Map with eviction at 200 entries |

The biggest culprit was likely `requestLog` — it captured every single request and never released any of them. Over days of traffic, that alone would cause steady memory growth. The `hits` Map in rate-limiter is fine since the existing `setInterval` cleanup handles it.
