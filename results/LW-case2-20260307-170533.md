Here's a summary of all changes:

**`src/services/order-service.js`**
- `listOrders`: both query and count now use `.whereNull('deleted_at')` instead of `.where('status', '!=', 'deleted')`
- `softDeleteOrder`: sets `deleted_at: db.fn.now()` instead of `status: 'deleted'`

**`src/routes/orders.js`**
- GET `/:id` and DELETE `/:id`: guard checks use `order.deleted_at` instead of `order.status === 'deleted'`

**`src/seed.js`**
- Soft-deleted seed orders now get a real business status (`pending`/`confirmed`/`shipped`) and a `deleted_at` timestamp instead of `status: 'deleted'`

**`src/utils/validators.js`**
- Updated comment to reflect that soft-delete uses `deleted_at` timestamp

Note: `src/services/user-service.js` already uses `.whereNull('deleted_at')` for its order queries, so no changes needed there. The `products` table uses an `is_active` boolean flag and has no `deleted_at` column — that's a separate migration if you want to standardize it.
