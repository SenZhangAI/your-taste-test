**Summary:** The JSDoc on `getOrder` claimed it filtered soft-deleted records, but the query was just `.where({ id }).first()` with no status filter. The route handler happened to have its own `status === 'deleted'` guard, which is why deletes appeared to work in some contexts — but any direct call to `getOrder` (including from `createOrder`) would return deleted records. The fix adds `.whereNot('status', 'deleted')` to the query so the behavior matches the documentation.
