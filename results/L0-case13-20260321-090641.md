Now `sort=price` maps to the `total_cents` integer column instead of silently falling back to `created_at`.

**What was happening:** `price` wasn't in `SORTABLE_FIELDS`, so the service fell back to sorting by `created_at`. The results appeared misordered by price because they were actually ordered by creation date. The `total_cents` column is an integer, so there was never a string comparison issue — the feature just didn't exist.

**What changed in `order-service.js`:**
- Added `'price'` to `SORTABLE_FIELDS` 
- Added `SORT_FIELD_MAP` to translate the user-facing `price` param to the actual DB column `total_cents`
