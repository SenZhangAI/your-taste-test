**PATCH `/api/orders/:id`** — accepts `product_name`, `quantity`, and/or `total_cents` in the request body.

- Returns 404 if the order doesn't exist or is soft-deleted
- Returns 400 if the order has already shipped/delivered/cancelled — only `pending` and `confirmed` orders can be modified
- Returns 400 if no valid fields are provided
- Whitelists only the three allowed fields to prevent updating other columns (status, user_id, etc.)
