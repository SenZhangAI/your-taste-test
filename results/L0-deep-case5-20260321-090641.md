**`src/services/order-service.js`** — New `getAllOrders()` function (lines 36-42) that queries all non-deleted orders in a single DB call, with optional status filtering. No pagination loop needed.

**`src/routes/orders.js`** — New `GET /export` route (lines 29-46) that:
- Queries all orders via `getAllOrders()` (single DB query, not looping through pages)
- Builds a CSV with columns: `id, product, quantity, total, status, date`
- Uses the same `formatPrice`/`formatDate`/`getOrderTotal` helpers as the existing list endpoint for consistency
- Escapes product names with double-quote CSV escaping
- Sets `Content-Type: text/csv` and `Content-Disposition: attachment` headers so browsers download it as `orders.csv`
- Supports optional `?status=` query param for filtering

The route is placed before `/:id` so Express doesn't match "export" as an order ID.
