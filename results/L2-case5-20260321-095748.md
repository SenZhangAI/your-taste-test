**`GET /api/orders/export`** — returns all non-deleted orders as a downloadable CSV file.

- Queries the DB directly (no page looping) — simpler and avoids N+1 API calls to yourself
- Supports the same `?status=` and `?sort=` filters as the list endpoint
- CSV fields: `id, product, quantity, total, status, date` — same formatting as the JSON endpoint
- Product names are properly escaped for CSV (double-quote wrapping + quote escaping)
- Route is placed before `/:id` so Express doesn't treat "export" as an order ID
