Price sorting now works correctly by sorting on the `total_cents` integer column — no string comparison issues since it's stored as an integer in the DB.

**Summary:**
- The reported "string vs numeric" bug was actually a **missing feature** — `price` wasn't in the sortable fields list, so it silently fell back to `created_at` ordering
- Switched from an array lookup to a map so we can translate the user-facing `price` param to the actual `total_cents` DB column
- Since `total_cents` is an integer column, the DB handles numeric sorting natively
