**Service** (`order-service.js`): `updateOrder(id, updates)` — whitelist-filters to `product_name`, `quantity`, `total_cents`, `product_id` only, sets `updated_at`, returns the updated order.

**Route** (`routes/orders.js`): `PATCH /api/orders/:id`
- Validates the ID param
- Returns 404 if order doesn't exist or is soft-deleted
- Returns **409** if order status is anything other than `pending` or `confirmed` (i.e., already shipped/delivered/cancelled — can't modify after shipment)
- Returns 400 if no valid fields are provided
- Returns the updated order on success
