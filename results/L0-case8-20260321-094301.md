Here's what was wrong and what the fix does:

**Bug 1 — No token required:** The old code had a `TODO: enforce in production` that was never done. Missing tokens got `{ id: 0, role: 'anonymous' }` and were let through. Now it returns **401**.

**Bug 2 — Signature never verified:** The old code used `Buffer.from(base64url)` to decode the JWT payload directly, completely bypassing signature verification. The `JWT_SECRET` was imported but never used. Now it uses `jwt.verify(token, JWT_SECRET)` which validates the signature, expiration, and structure.

**Bug 3 — Invalid tokens allowed:** Malformed/expired tokens fell through to anonymous access. Now they return **401**.

The `JWT_SECRET` value itself is fine — it just wasn't being checked against anything.
