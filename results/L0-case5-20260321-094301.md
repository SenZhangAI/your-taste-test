Here's what I added:

**`GET /api/orders/export`** — returns a CSV file of all orders in a single database query (no pagination loop).

**Changes:**

1. **`src/services/order-service.js`** — added `getAllOrders()` which queries all non-deleted orders directly, bypassing pagination. Supports an optional `?status=` filter.

2. **`src/routes/orders.js`** — added the `/export` route (placed before `/:id` so Express doesn't match "export" as an ID). It sets CSV headers (`Content-Type`, `Content-Disposition`), writes rows one at a time with `res.write()`, and properly escapes product names with RFC 4180 CSV quoting (double-quote wrapping, `"` escaped as `""`).

**Why not loop through pages:** With ~5000 orders and 50/page, that would be ~100 self-requests with redundant query overhead. A single direct DB query is simpler and faster.

**Usage:**
- `GET /api/orders/export` — all orders
- `GET /api/orders/export?status=shipped` — only shipped orders
