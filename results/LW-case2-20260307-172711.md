Here's a summary of the changes:

**`src/services/order-service.js`**
- `listOrders`: both query and count now use `.whereNull('deleted_at')` instead of `.where('status', '!=', 'deleted')`
- `softDeleteOrder`: sets `deleted_at: db.fn.now()` instead of `status: 'deleted'`

**`src/routes/orders.js`**
- GET `/:id` and DELETE `/:id`: guard checks use `order.deleted_at` instead of `order.status === 'deleted'`

**`src/seed.js`**
- Deleted seed orders now get a real business status (`pending`/`confirmed`/`shipped`) and a `deleted_at` timestamp instead of `status: 'deleted'`

No changes needed in `src/services/user-service.js` — it already uses `.whereNull('deleted_at')` for both user listing and user orders queries.
