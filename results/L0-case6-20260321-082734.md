**The bug:** `getOrder` (`src/services/order-service.js:48`) had no filter on soft-deleted records despite its JSDoc claiming otherwise. It was just `db('orders').where({ id }).first()`.

The route handlers in `orders.js` had a secondary check (`order.status === 'deleted'`) on lines 32 and 82, which masked the bug for the `GET /:id` and `DELETE /:id` endpoints. But if `getOrder` was being called from anywhere else (or if that route-level check was ever removed), deleted orders would leak through.

The fix adds `.whereNot('status', 'deleted')` so the function actually does what it says — filtering deleted records at the data layer where it belongs.
