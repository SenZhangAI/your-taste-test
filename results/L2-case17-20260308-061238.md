**`PATCH /api/orders/:id`** — updates an order with these rules:

- **Allowed fields**: `product_name`, `product_id`, `quantity`, `total_cents` (whitelist — ignores anything else)
- **Blocked after shipment**: returns `409` if order status is `shipped` or `delivered`
- **Validation**: `total_cents` and `quantity` must be positive; at least one valid field required
- **Deleted/missing orders**: returns `404`

The service layer (`updateOrder`) is generic — takes an id and a fields object, sets `updated_at` automatically.
