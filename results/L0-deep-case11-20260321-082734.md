Here's what was added:

**`order-service.js`** — `createOrder` now:
1. Looks up the product by `product_name` in the `products` table
2. Returns a 400 error if the product doesn't exist
3. Returns a 409 error if `stock < requested quantity`
4. Decrements `stock` by the order quantity before inserting the order
5. Also populates `product_id` from the lookup if it wasn't provided

**`routes/orders.js`** — The POST handler now catches errors from `createOrder` and returns the appropriate status code and message.
