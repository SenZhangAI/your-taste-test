Here's what was added:

**In `order-service.js`** — `createOrder` now:
1. Looks up the product by `product_name` in the `products` table
2. Returns a **400** if the product isn't found
3. Returns a **409** if `product.stock < quantity`
4. Deducts `quantity` from the product's `stock` before inserting the order
5. Also auto-populates `product_id` from the looked-up product if not provided

**In `routes/orders.js`** — the POST handler catches these errors and returns the appropriate status code and message.
