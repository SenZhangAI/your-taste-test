Two bugs fixed:

1. **No token → 401 rejection** instead of silently assigning anonymous access. The `// TODO: enforce in production` was never done — requests without tokens were waved through.

2. **`jwt.verify()` replaces manual `Buffer.from()` decode** — tokens are now cryptographically verified against `JWT_SECRET`. Previously any base64-encoded JSON payload was accepted as a valid identity, signature completely ignored.

The `JWT_SECRET` env var you already have set will now actually be used for verification.
