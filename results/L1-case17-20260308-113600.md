**`PATCH /api/orders/:id`** — updates an order's `product_name`, `product_id`, `quantity`, and/or `total_cents`.

Key behaviors:
- **Only pre-shipment orders** (`pending`/`confirmed`) can be modified. Orders in `shipped`, `delivered`, or `cancelled` status return `409 Conflict`.
- **Field whitelist** — only the 4 updatable fields are accepted; everything else is silently ignored.
- **Validation** — invalid ID → `400`, order not found/deleted → `404`, no valid fields in body → `400`.
- `updated_at` is automatically set on every update.
