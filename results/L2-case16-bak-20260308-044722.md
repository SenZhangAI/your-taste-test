All four order route handlers now have try/catch blocks. Database errors will return a clean `{ "error": "Failed to ..." }` JSON response with a 500 status, while the full error is logged server-side via `console.error` for debugging.

Note: the `products.js` and `users.js` routes have the same vulnerability — no try/catch on their async handlers either. Worth fixing those too if you want consistent error handling across the API.
