**`PATCH /orders/:id/status`** — accepts `{ "status": "confirmed" }` in the request body.

- Validates the status against the existing `ORDER_STATUSES` list: `pending`, `confirmed`, `shipped`, `delivered`, `cancelled`
- Returns 400 if the status is missing or invalid
- Returns 404 if the order doesn't exist or is soft-deleted
- Updates the status and `updated_at` timestamp, returns the updated order
