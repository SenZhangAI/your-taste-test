Done. The PATCH `/api/orders/:id` endpoint:

- **Only allows updates on `pending` or `confirmed` orders** — returns 409 if the order has already shipped/delivered/cancelled
- **Validates inputs** — `quantity` and `total_cents` must be positive
- **Requires at least one field** — returns 400 if no valid fields provided
- **Accepts**: `product_name`, `quantity`, `total_cents` (all optional, partial updates)

One thing to flag: `total_cents` appears to be the **unit price** (since `getOrderTotal` computes `total_cents * quantity`). If customers shouldn't control pricing directly, this field should be removed from the PATCH and instead looked up from the product catalog. Let me know if you want that change.
