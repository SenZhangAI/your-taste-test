**The bug:** The count query (line 25) was building a fresh `db('orders')` query instead of reusing the already-filtered `query` variable. So when a user filtered by status (e.g., `?status=pending`), the data query correctly returned only matching orders, but the total count always returned the count of *all* non-deleted orders.

**The fix:** Changed the count query to use `query.clone()` so it applies the same filters (including the status filter) as the data query. Now the total will correctly reflect only the filtered results.
