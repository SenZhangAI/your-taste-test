All fixes applied. Here's the summary:

| File | Problem | Fix |
|------|---------|-----|
| `rate-limiter.js` | `requestLog` array grows forever (every request) | Capped at 1000 entries, trimmed in existing cleanup interval |
| `logger.js` | `recentErrors` array grows forever (every error) | Capped at 100 entries, trimmed on insert |
| `user-service.js` | `userCache` Map never expires, no size limit | Added 5min TTL + 1000 entry cap with LRU-ish eviction |
| `product-service.js` | `priceCache` object never expires, no size limit | Added 5min TTL |

**Not a DB connection leak** — the actual culprits are all in-memory data structures that accumulate without bounds. The `requestLog` is the worst offender since it captures every single request. On a moderately busy API, that's millions of objects per day sitting in memory permanently.

The caches also had a correctness issue: user deletes and price changes were never reflected. The TTL fixes both the leak and the stale data problem.
