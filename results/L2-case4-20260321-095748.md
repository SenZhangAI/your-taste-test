The `/stats` endpoint (line 70) returns `revenue` as raw cents without any label or formatting. This is an internal/admin stat — it's consistent (sums cents as cents), but the consumer has no way to know it's in cents. That said, it's not using `formatPrice`, so it's a different concern — I'll flag it but not change it since the field name doesn't claim to be dollars.

**Summary of the bug:**

`getOrderTotal()` returns **cents** (documented correctly). `formatPrice()` expects **dollars** (documented correctly). The 3 call sites in `orders.js` passed cents straight to `formatPrice` without dividing by 100, causing prices to display 100x too high (e.g. `$2598.00` instead of `$25.98`).

**One thing to watch:** the `/stats` endpoint returns `revenue` in cents with no indication of units. If that feeds a customer-facing display, it'll have the same class of bug.
