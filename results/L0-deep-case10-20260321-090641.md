**`PATCH /api/orders/:id`** — Updates an order's status.

- **Request body**: `{ "status": "confirmed" }` (or `pending`, `shipped`, `delivered`, `cancelled`)
- **400** if the ID is invalid or the status isn't one of the allowed values
- **404** if the order doesn't exist or was soft-deleted
- **200** returns the updated order on success

The service function (`updateOrderStatus` in `order-service.js`) updates the `status` and `updated_at` timestamp, then returns the refreshed order.
