**What changed:**
- Added `invalidatePriceCache(productId)` to `product-service.js` — deletes the cached entry for a given product
- Called it in `PATCH /products/:id` right after the DB update, so the next `getProductPrice()` call fetches the fresh price from the DB

**Broader note:** This cache has no TTL and lives forever in memory — it was designed for "batch operations" per the comment, but it's being used as a general-purpose lookup cache. If this service ever runs in multiple processes (cluster mode, multiple containers), this in-memory cache will also be stale across instances. Worth considering whether this cache is needed at all, or if it should be replaced with a proper cache (Redis, TTL-based) if performance is a concern.
