**`PATCH /api/orders/:id`** — Updates an order's fields before shipment.

**Route** (`src/routes/orders.js`):
- Validates the ID with `validateId`
- Returns 404 if the order doesn't exist or is deleted
- Returns **409 Conflict** if the order status is anything other than `pending` or `confirmed` (i.e., already shipped/delivered/cancelled)
- Validates `status`, `total_cents`, and `quantity` if provided
- Returns the updated order formatted consistently with the GET endpoint

**Service** (`src/services/order-service.js`):
- `updateOrder(id, fields)` — Whitelists updatable fields (`product_name`, `product_id`, `quantity`, `total_cents`, `status`), sets `updated_at`, and returns the updated order
