All four leaks fixed. Here's a summary:

| File | Problem | Fix |
|------|---------|-----|
| `src/middleware/rate-limiter.js` | `requestLog` array grew with every request, never cleaned | Replaced with a simple `requestCount` counter |
| `src/middleware/logger.js` | `recentErrors` array grew with every error response, never trimmed | Cap at 50 entries using `shift()` when full |
| `src/services/user-service.js` | `userCache` Map grew per unique user, never evicted | Added 10-minute TTL expiry |
| `src/services/product-service.js` | `priceCache` object grew per unique product, never evicted | Converted to Map with 5-minute TTL |

The first two (rate-limiter and logger) were the primary culprits — they grow linearly with traffic volume and would accumulate thousands of objects per hour. The caches grow more slowly (bounded by unique entity count) but still had no eviction, meaning stale data would also accumulate indefinitely.
