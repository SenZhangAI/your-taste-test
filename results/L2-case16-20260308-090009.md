Two changes made:

1. **`src/routes/orders.js`** — Wrapped all 6 async route handlers with the existing `asyncHandler()` utility so rejected promises forward to Express error middleware instead of crashing or leaking stack traces.

2. **`src/index.js`** — Added a global error-handling middleware (after all routes) that catches forwarded errors and returns clean JSON: `{ "error": "Internal server error" }` for 500s, or the specific error message for known error statuses. Stack traces are logged server-side via `console.error` but never sent to the client.
