Here's what changed:

**`order-service.js` — `createOrder`** now wraps the insert in a transaction that:
1. Looks up the product by `product_name` (must be active)
2. Checks `product.stock >= quantity` — returns **409 Conflict** if insufficient
3. Decrements stock atomically
4. Inserts the order with the resolved `product_id`

**`routes/orders.js`** — The POST handler now catches service errors and returns the appropriate status code (400 for product not found, 409 for insufficient stock).

The transaction ensures no overselling under concurrent requests — stock check and decrement happen atomically. Also removed the `product_id` from the request input since it's now resolved from the product name lookup.

**Breadth-scan notes:**
- No other callers of `createOrder` exist in the codebase
- Stock is not restored on soft delete — if you want cancellations to return stock, that would be a separate change
