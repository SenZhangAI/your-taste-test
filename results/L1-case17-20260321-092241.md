**`PATCH /api/orders/:id`** — updates an order before shipment.

**Route** (`src/routes/orders.js:79-110`):
- Validates `:id` with `validateId`
- Validates `quantity` and `total_cents` are positive integers when provided
- Accepts: `product_name`, `product_id`, `quantity`, `total_cents` (all optional, at least one required)
- Returns formatted order on success

**Service** (`src/services/order-service.js:74-96`):
- Whitelists updatable fields via `UPDATABLE_FIELDS` — prevents injection of `status`, `user_id`, etc.
- Only allows updates on `pending` or `confirmed` orders — shipped/delivered/cancelled/deleted orders return a 400
- Sets `updated_at` on every update

**Error responses:**
- `400` — invalid ID, invalid field values, no fields provided, or order already shipped+
- `404` — order not found or deleted
