**Fixed.** `getOrder` now filters out deleted records at the service layer (`order-service.js:28`), matching its JSDoc contract.

**Summary:**
- **Root cause:** `getOrder` performed `db('orders').where({ id }).first()` with no deleted filter, despite the JSDoc claiming otherwise. The misleading comment made it look safe, but the actual query returned all orders regardless of status.
- **Why the route check wasn't enough:** The route handler (`orders.js:28`) did check `order.status === 'deleted'`, but relying solely on the caller to filter is fragile — any new code calling `getOrder` (including `createOrder` on line 47) would bypass that check.
- **Fix:** Added `.whereNot('status', 'deleted')` to the query in `getOrder`, consistent with how `listOrders` already filters on line 11.
