Fixed all four leaks:

**Critical fixes:**
- **`requestLog[]`** (rate-limiter.js) — capped at 1000 entries, splices oldest half when full
- **`recentErrors[]`** (logger.js) — capped at 100 entries, same splice strategy

**Moderate fixes:**
- **`userCache`** (user-service.js) — LRU eviction with 500 entry cap (Map insertion order = natural LRU)
- **`priceCache`** (product-service.js) — switched from plain object to Map with 5-min TTL. Bonus: price updates now actually take effect within 5 minutes instead of being stale forever

Not a DB connection leak — Knex connection pooling is fine. The memory growth was from these four unbounded in-memory data structures, with `requestLog` being the biggest contributor (one entry per request, never cleaned).
