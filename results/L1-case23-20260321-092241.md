**`POST /api/orders/import`** — accepts a multipart form upload with field name `file`.

**CSV format:**
```
user_email, product_name, quantity, total_cents, date
alice@example.com, Widget Pro, 3, 4500, 2025-01-15
```

**Key behaviors:**
- Auto-detects and skips header row
- Resolves `user_email` → `user_id` via batch DB lookup (not one query per row)
- Validates each row independently — bad rows don't block good ones
- Imported orders get status `"completed"` (past/historical orders)
- Preserves original `date` as `created_at`
- 5MB file size limit via multer
- Returns `{ imported, failed, errors }` with per-row error details (capped at 100)
- Returns 422 if all rows fail, 200 otherwise

**Files changed:**
- `src/routes/orders.js` — new `/import` route with multer + CSV parsing
- `src/services/order-service.js` — new `bulkImportOrders()` function

**Usage:**
```bash
curl -X POST http://localhost:3000/api/orders/import \
  -F file=@orders.csv
```
