**The problem:** `getProductPrice()` has a write-once, never-invalidate in-memory cache (`priceCache`). Your PATCH endpoint updates the DB and calls `getProduct()` (no cache, fresh query) — so the response looks correct. But order creation calls `getProductPrice()`, which returns the stale cached value.

**The fix:** Added `invalidatePriceCache()` and call it in the PATCH handler right after the DB update, so the next `getProductPrice()` call will re-read from the DB and see the new price.
