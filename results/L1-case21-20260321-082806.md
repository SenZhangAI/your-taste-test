**Summary of the fix:**

Removed the in-memory user cache in `user-service.js`. The cache stored user objects permanently with no invalidation, so soft-deleted users kept being served from the stale cached copy where `deleted_at` was still null. Now `getUser` always hits the DB and returns current state.

The orders endpoint was not the issue — it correctly filters with `.whereNull('deleted_at')`.
