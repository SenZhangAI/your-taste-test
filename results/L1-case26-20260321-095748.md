**Root cause:** `priceCache` is a module-level object with no invalidation. Once `getProductPrice()` is called for a product, the price is cached in memory forever. The `PATCH /products/:id` endpoint updates the DB but doesn't (and can't cleanly) clear this cache, so subsequent `getProductPrice()` calls return stale data.

**Fix:** Removed the cache entirely. A single `SELECT ... WHERE id = ?` per order creation is negligible — the cache was a premature optimization that traded correctness for a DB round-trip that didn't need saving.

**Breadth-scan:** No other in-memory caches in the codebase. The price update endpoint in `routes/products.js` correctly reads fresh data via `getProduct()` (which was never cached), which is why the update response showed the correct price while order creation didn't.
