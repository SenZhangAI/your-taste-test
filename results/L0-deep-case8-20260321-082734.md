Here's what changed:

- **Missing token**: Now returns `401` instead of falling through as anonymous
- **Token validation**: Uses `jwt.verify(token, JWT_SECRET)` which validates the signature, expiration, and structure — instead of just base64-decoding the payload
- **Invalid token**: Now returns `401` instead of falling through as anonymous

The `JWT_SECRET` env var you already have set in prod will now actually be used to cryptographically verify tokens. Previously it was imported but completely unused.
