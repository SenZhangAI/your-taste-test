Changes made:

- **Missing token now returns 401** instead of silently allowing anonymous access
- **`jwt.verify()` replaces manual base64 decode** — signature is now validated against `JWT_SECRET`, and expiration (`exp`) is checked automatically
- Invalid/expired tokens return 401 instead of falling through as anonymous

The `JWT_SECRET` env var you already have set in prod will now actually be used for verification. Public paths (`/health`, `/api/products`) remain open as before.
