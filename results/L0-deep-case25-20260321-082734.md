The fix: the total count query on line 25 was building a **new** query against `db('orders')` that only excluded deleted orders, instead of reusing the `query` variable which already includes the status filter. So when filtering by e.g. `status=pending`, you'd get the correct 3 pending orders but the total would reflect all 47 non-deleted orders.

Changed it to `query.clone().count(...)` so the count respects the same filters applied to the data query.
