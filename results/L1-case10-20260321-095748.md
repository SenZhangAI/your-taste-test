**`PATCH /api/orders/:id/status`** — Updates an order's status with transition validation.

**Request:** `{ "status": "confirmed" }`

**Validation layers:**
1. **ID validation** — rejects non-numeric/invalid IDs (400)
2. **Status validation** — must be one of the known `ORDER_STATUSES` (400)
3. **Transition validation** — enforces a state machine (422):
   - `pending` → `confirmed` | `cancelled`
   - `confirmed` → `shipped` | `cancelled`
   - `shipped` → `delivered`
   - `delivered`, `cancelled` → terminal (no further changes)
4. **Existence check** — 404 for missing/deleted orders

**Response:** Returns the updated order object on success, or `{ error: "..." }` with appropriate status code on failure.
