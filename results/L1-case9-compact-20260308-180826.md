Found and fixed 4 memory leaks:

| Source | Problem | Fix |
|--------|---------|-----|
| `rate-limiter.js` `requestLog` | **#1 culprit** — every request appends an object, never cleaned | Cap at 1000 entries, prune when doubled |
| `logger.js` `recentErrors` | Every 4xx/5xx appends forever, only last 50 ever read | Prune to 50 when it hits 100 |
| `user-service.js` `userCache` | Map grows with every unique user, never evicts | LRU eviction, capped at 500 |
| `product-service.js` `priceCache` | Object grows with every product lookup, never clears | Clear-all when hitting 500 |

The `requestLog` is almost certainly your main leak — at ~200-500 bytes per entry, a server handling 100 req/s accumulates ~1-2 GB/day from that array alone. The other three are slower leaks but still unbounded.

Not a connection leak (better-sqlite3 is synchronous, single connection) — it's all in-memory data structures with no caps.
