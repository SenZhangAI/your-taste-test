Here's what was added:

1. **Product lookup by name** — finds the product from `product_name` on the order
2. **Stock check** — rejects with 409 if requested quantity exceeds available stock
3. **Transactional decrement** — uses a DB transaction with a `WHERE stock >= qty` guard to atomically decrement stock and insert the order, preventing race conditions
4. **Auto-links `product_id`** — if not provided, it's filled from the looked-up product

The route now catches these errors and returns the appropriate status code (400 for unknown product, 409 for insufficient stock).
