The LIKE query is fine — the issue is **Express route ordering**. `/:id` was defined before `/search`, so `GET /orders/search` matched `/:id` with `id = "search"`, returned no order, and gave you the 404.

Same issue applies to `/stats` (line 66) — it was also below `/:id` and would have the same problem. Both `/search` and `/stats` are now above `/:id`.
