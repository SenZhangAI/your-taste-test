Changes made:

1. **POST `/api/orders`** — now returns formatted `total` and `date` fields matching GET's shape
2. **Fixed cents/dollars bug in all 4 handlers** (GET list, GET single, POST, search) — `getOrderTotal` returns cents, so dividing by 100 before passing to `formatPrice` which expects dollars

Without the `/100`, an order with `total_cents: 2999, quantity: 1` would display as `$2999.00` instead of `$29.99`.
