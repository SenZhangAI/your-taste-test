Here's a summary of all changes:

**Bug fix — `softDeleteOrder` (`src/services/order-service.js`):**
- Now sets `deleted_at: db.fn.now()` alongside `status: 'deleted'`. This fixes the latent bug where `getUserOrders()` in user-service.js filters by `whereNull('deleted_at')` — soft-deleted orders were still appearing in user order lists.

**Products aligned to `deleted_at` pattern:**
- **New migration** (`migrations/004-add-deleted-at-to-products.js`): Adds `deleted_at` column to products, migrates existing `is_active=false` rows.
- **`src/services/product-service.js`**: `listProducts()` now filters by `whereNull('deleted_at')` instead of `is_active`. Added `softDeleteProduct()` which sets both `is_active=false` and `deleted_at` (keeping `is_active` in sync for backward compatibility).
- **`src/routes/products.js`**: GET `/:id` now checks `product.deleted_at` instead of `!product.is_active`. Added `DELETE /:id` endpoint using `softDeleteProduct`.
- **`src/db.js`**: Products schema now includes `deleted_at` nullable timestamp.

All three entities (users, orders, products) now consistently use `deleted_at` timestamps for soft-delete filtering.
