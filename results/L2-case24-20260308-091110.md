Changes look good. Here's what was added:

**Migration** (`004-add-refund-to-orders.js`): Adds `refund_cents` (unsigned int, default 0) and `cancelled_at` (nullable timestamp) to orders.

**Service** (`cancelOrder`): Validates refund doesn't exceed order total, sets status to `cancelled`, records refund amount and cancellation time.

**Endpoint** (`POST /api/orders/:id/cancel`):
- Request body: `{ "refund_cents": 500 }` (supports partial refunds)
- Validates: valid ID, order exists, not already cancelled, refund ≤ order total
- Returns the cancelled order with formatted `refund` and `cancelled_at`

**Order responses**: `GET /:id` and `GET /` now include `refund` and `cancelled_at` fields when the order is cancelled.
