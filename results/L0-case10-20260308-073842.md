**`PATCH /orders/:id/status`** — Updates an order's status. Send `{ "status": "confirmed" }` in the request body.

- Validates that `status` is one of the allowed values: `pending`, `confirmed`, `shipped`, `delivered`, `cancelled`
- Returns 400 if the status is missing or invalid
- Returns 404 if the order doesn't exist or is soft-deleted
- Returns the updated order on success
