**Root cause:** Backend bug in `order-service.js:25-27`. The count query was building a fresh `db('orders').where('status', '!=', 'deleted')` instead of reusing the `query` variable that already includes the status filter. So the total always reflected all non-deleted orders regardless of the active filter.

**Fix:** Reuse `query.clone()` for the count, which carries the same `status` filter as the data query.

**Breadth-scan:** `searchOrders` (line 35-41) doesn't return a total count at all — it just returns up to 50 results with no pagination info. That's a separate concern but worth noting if search also needs pagination later.
