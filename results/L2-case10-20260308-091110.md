Done. Added `PATCH /orders/:id/status` with:

- **Request body:** `{ "status": "confirmed" }` — must be a valid status from `ORDER_STATUSES`
- **Transition enforcement:** only forward transitions allowed:
  - `pending` → `confirmed` / `cancelled`
  - `confirmed` → `shipped` / `cancelled`
  - `shipped` → `delivered`
  - `delivered` / `cancelled` → terminal, no further changes
- **Error responses:** 400 for invalid status or illegal transition, 404 for missing/deleted orders
