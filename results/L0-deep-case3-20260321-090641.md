Here's what I changed and why:

1. **`src/config.js`** — `RATE_LIMIT` now reads from `process.env.RATE_LIMIT` with a default of 500. This makes it consistent with how `PORT` and `JWT_SECRET` already work, and means the docs/`.env.example` aren't lying anymore.

2. **`.env.example`** — Updated from `10` to `500` to reflect the new default.

Previously, no matter what you set in `.env`, the rate limit was always 100 because the env var was never read. Now it's actually configurable — and if users need a different value in the future, they can set it in their `.env` without a code change.
