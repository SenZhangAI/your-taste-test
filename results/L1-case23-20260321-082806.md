**`POST /api/orders/import`** — accepts a multipart form with a `file` field containing a CSV.

**Route** (`src/routes/orders.js:46-76`):
- multer with memory storage, 5MB limit
- Validates file presence, header row with required columns
- Parses CSV → row objects, delegates to service
- Returns 400 if all rows fail, 200 otherwise (partial success is OK)

**Service** (`src/services/order-service.js:76-138`):
- Batch-loads all referenced emails in one query (no N+1)
- Per-row validation: email→user resolution, required fields, positive integers, valid date
- Inserts valid rows with status `'completed'` and `created_at` from CSV date
- Batches inserts (100/batch) to stay within SQLite variable limits
- Returns `{ imported, failed, errors: [{ row, errors }] }` for actionable feedback

**Example usage:**
```
curl -X POST http://localhost:3000/api/orders/import \
  -H "Authorization: Bearer <token>" \
  -F "file=@orders.csv"
```

**CSV format:**
```
user_email,product_name,quantity,total_cents,date
alice@example.com,Widget Pro,2,4999,2024-01-15
bob@example.com,Gadget X,1,2999,2024-02-20
```
