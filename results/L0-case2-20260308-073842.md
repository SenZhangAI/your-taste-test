Here's a summary of the changes:

**`src/services/order-service.js`:**
- `listOrders` — both the query and count now use `.whereNull('deleted_at')` instead of `.where('status', '!=', 'deleted')`
- `softDeleteOrder` — sets `deleted_at: db.fn.now()` instead of `status: 'deleted'`

**`src/routes/orders.js`:**
- GET `/:id` and DELETE `/:id` — check `order.deleted_at` instead of `order.status === 'deleted'`

Note that `getOrder` (line 32) fetches by ID without any soft-delete filter — it relies on the caller to check. That's unchanged, and the route handlers now guard on `deleted_at` instead of status. If you'd prefer `getOrder` to also filter out deleted rows at the query level, let me know.
