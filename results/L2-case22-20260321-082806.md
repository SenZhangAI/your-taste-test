**Root cause:** Express matches routes in registration order. `GET /:id` was defined before `GET /search` and `GET /stats`, so requests to `/search` and `/stats` were being captured by the `:id` wildcard param — leading to "Order not found" because no order has id `"search"`.

**Fix:** Moved both `/search` and `/stats` above `/:id`. General rule: static path segments must be registered before parameterized ones on the same path level.
