Done. Here's what was added:

**`src/middleware/auth.js`** — new `impersonate` middleware:
- Checks for `?as_user=<id>` query param on any endpoint
- Rejects with 403 if caller is not `role: 'admin'`
- Validates the ID is a positive integer
- Looks up the target user in DB, rejects 404 if not found or soft-deleted
- Swaps `req.user` to the target user, stores original admin as `req.adminUser` for audit trail

**`src/index.js`** — registered `impersonate` after `authenticate`, before `rateLimiter`

Usage: `GET /api/orders?as_user=123` with an admin JWT will execute the request as user 123. All downstream code reading `req.user` sees the impersonated user. The original admin identity is available on `req.adminUser` for logging/audit if needed.
