Done. The `listOrders` query now joins the `users` table to fetch `users.name as user_name`, and the orders list response includes a `user` field with the user's name. Each order response now looks like:

```json
{
  "id": 1,
  "user": "John Doe",
  "product": "Widget",
  "quantity": 2,
  "total": "$19.98",
  "status": "pending",
  "date": "..."
}
```

The join is done at the query level (single DB call) rather than N+1 lookups per order. The count query for pagination is unaffected since it only needs the orders table.
