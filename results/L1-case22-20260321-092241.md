**Root cause:** Express matches routes top-down. `/:id` was declared before `/search` and `/stats`, so requests to those paths matched `:id` as a parameter (with values `"search"` and `"stats"`), hit `getOrder()`, got null, and returned 404.

**Fix:** Moved all literal path routes (`/search`, `/stats`) above the parameterized `/:id` route. The route order is now:

1. `GET /` — list
2. `GET /search` — search
3. `GET /stats` — stats
4. `GET /:id` — single order (catch-all, must be last)
5. `POST /` — create
6. `DELETE /:id` — soft delete
