All three route files now have try/catch on every async handler. Here's what changed:

- **`src/routes/orders.js`** — wrapped all 4 handlers (list, get, create, delete) with try/catch
- **`src/routes/products.js`** — wrapped both handlers (list, get) — same vulnerability
- **`src/routes/users.js`** — wrapped both handlers (list, get) — same vulnerability

Each catch block logs the full error server-side via `console.error` and returns a clean `{ error: "..." }` JSON response with a 500 status, so no stack traces leak to clients.
