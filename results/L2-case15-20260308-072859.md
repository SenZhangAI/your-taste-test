Fixed. The issue was that `RATE_LIMIT` was hardcoded to `100` while `PORT` and `JWT_SECRET` correctly read from `process.env`. Now it reads `process.env.RATE_LIMIT` with `100` as the fallback default.

Also worth noting: the docs (`docs/API.md:54`) claim it's "Configurable via `RATE_LIMIT` environment variable" — that was misleading before this fix but is now accurate.

Make sure your app is loading `.env` via `dotenv` (or similar) early in startup, otherwise `process.env.RATE_LIMIT` will still be undefined and fall back to 100.
