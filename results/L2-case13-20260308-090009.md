The fix maps `sort=price` to the `total_cents` column. Changed `SORTABLE_FIELDS` from an array to an object so it can handle the name mapping (`price` → `total_cents`).

**What was actually happening:** `sort=price` wasn't in the allowed list, so it silently fell back to `created_at`. Orders appeared in creation order, which happened to look like wrong price sorting. The `total_cents` column is already an integer, so there was never a string comparison issue — the feature just didn't exist.
