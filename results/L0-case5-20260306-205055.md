The endpoint is ready. `GET /api/orders/export` will:

1. Fetch all non-deleted orders via the new `getAllOrders()` service function
2. Format them as CSV with columns: `id, product, quantity, total, status, date`
3. Return the response with `text/csv` content type and a download filename of `orders.csv`

The `/export` route is placed before `/:id` so Express doesn't treat "export" as an order ID.
