The fix maps `sort=price` to the `total_cents` DB column, so sorting happens numerically at the database level.

**Root cause**: `SORTABLE_FIELDS` didn't include price, so `sort=price` silently fell back to `created_at` ordering. The results looked like string-sorted prices, but they were actually just creation-time ordered.

**What changed**: Replaced the allowlist array with a `SORT_FIELD_MAP` that maps API parameter names to DB columns, adding `price → total_cents`. This keeps the DB sort (numeric, correct) rather than doing any post-query string comparison.
