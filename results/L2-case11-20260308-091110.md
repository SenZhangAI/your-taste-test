Done. Here's what the stock check does:

1. **Looks up the product by `product_name`** — returns 400 if not found
2. **Validates stock** — returns 409 (Conflict) if `stock < quantity`
3. **Uses a transaction** with an atomic `WHERE stock >= qty` guard on the UPDATE — prevents race conditions where two concurrent orders could both pass the initial check but oversell

The stock is decremented within the same transaction as the order insert, so they either both succeed or both roll back.
