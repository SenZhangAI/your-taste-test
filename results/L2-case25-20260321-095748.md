The bug: the count query on line 25 was building a **new query** that only excluded deleted orders, ignoring the status filter. So `total` always returned the count of all non-deleted orders regardless of the filter applied.

The fix: reuse `query` (which already has the status filter applied) for the count, via `.clone().count(...)`. Now filtering by `pending` will return both matching orders **and** the correct total for that filter.
