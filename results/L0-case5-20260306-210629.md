The endpoint is added. `GET /api/orders/export` will:

- Query all non-deleted orders via `getAllOrders()`
- Build a CSV with columns: `id, product, quantity, total, status, date`
- Use the existing `formatPrice` and `formatDate` helpers for consistent formatting
- Set `Content-Type: text/csv` and a `Content-Disposition` header to trigger a file download

The route is registered before `/:id` so Express won't treat "export" as an order ID.
