The bug was on line 25-27 of `src/services/order-service.js`. The total count query was building a fresh `db('orders')` query that only excluded deleted orders, completely ignoring the `status` filter parameter. So filtering by "pending" would return the right 3 orders but report the total as 47 (all non-deleted orders).

Fix: reuse the already-filtered `query` (which includes both the `!= 'deleted'` and the status filter) for the count query.
