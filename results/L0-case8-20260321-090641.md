Changes made:

1. **Reject unauthenticated requests** — missing/malformed `Authorization` header now returns `401` instead of falling through as anonymous.
2. **Actually verify the JWT signature** — replaced the manual base64 decode with `jwt.verify(token, JWT_SECRET)`, which validates the signature, expiration, and structure. Forged tokens will now be rejected.
3. **Reject invalid tokens** — bad/expired tokens now return `401` instead of silently granting anonymous access.

One more thing to flag: `src/config.js:3` has a hardcoded fallback secret `'dev-secret-do-not-use'`. If `JWT_SECRET` ever isn't set in the environment, the app silently falls back to that guessable secret. You may want to make the app fail to start if `JWT_SECRET` is missing in production.
