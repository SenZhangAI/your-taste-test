Two bugs fixed in `src/middleware/auth.js`:

1. **No token → 401 now.** Previously, missing tokens were silently accepted with an "anonymous" user object, so every unauthenticated request got through. Now returns `401 Missing authentication token`.

2. **Signature verification.** Previously the code only base64-decoded the JWT payload without checking the signature — anyone could craft a fake token. Now uses `jwt.verify(token, JWT_SECRET)` which validates the signature against your `JWT_SECRET`.

The secret itself isn't the problem — it was never being used at all. The `jsonwebtoken` package was already in `package.json` but never imported.
