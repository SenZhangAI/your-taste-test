**`PATCH /api/orders/:id/status`** — updates order status with transition validation.

Request: `{ "status": "confirmed" }`

Responses:
- `200` — updated order object
- `400` — invalid status value
- `404` — order not found / soft-deleted
- `409` — invalid transition, with `allowed` array showing valid next states

State machine:
```
pending → confirmed, cancelled
confirmed → shipped, cancelled
shipped → delivered
delivered → (terminal)
cancelled → (terminal)
```

Used `409 Conflict` for invalid transitions since it's a business rule violation on current resource state, not a validation error.
