Now clients can pass `sort=total_cents` and get proper numeric sorting at the DB level. The `total_cents` column is `integer` in the schema, so the database handles numeric ordering correctly.

**Summary:** `SORTABLE_FIELDS` only allowed `created_at` and `updated_at`. Added `total_cents` so price sorting uses the DB's integer comparison instead of falling through to a default or being done on formatted strings client-side.
