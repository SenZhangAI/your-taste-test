**Service** (`src/services/order-service.js`): `updateOrder(id, updates)` — whitelists `product_name`, `quantity`, `total_cents`, `product_id`, and `status` to prevent mass-assignment. Sets `updated_at` automatically.

**Route** (`src/routes/orders.js`): `PATCH /api/orders/:id`
- **404** if order doesn't exist or is deleted
- **409** if order status is anything beyond `confirmed` (shipped/delivered/cancelled) — prevents modification after shipment
- **400** if an invalid status is provided or no valid fields are in the body
- Returns the updated order with formatted total and date, matching the GET response shape
