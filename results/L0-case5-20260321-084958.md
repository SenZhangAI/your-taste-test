Here's what I added:

**`GET /api/orders/export`** — returns all orders as a downloadable CSV file.

**Changes:**

1. **`src/services/order-service.js`** — added `getAllOrders()` which queries all non-deleted orders without pagination (supports optional `sort` and `status` filters, same as the list endpoint).

2. **`src/routes/orders.js`** — added the `/export` route (placed before `/:id` so Express doesn't treat "export" as an ID). It:
   - Fetches all orders via `getAllOrders()`
   - Formats them with the same columns as the JSON endpoint (id, product, quantity, total, status, date)
   - Properly escapes product names containing commas/quotes for CSV
   - Sets `Content-Type: text/csv` and `Content-Disposition: attachment` headers

**Query params** (optional, same as the list endpoint):
- `?status=shipped` — filter by status
- `?sort=updated_at` — sort field

Instead of looping through pages, it queries the DB directly — simpler, faster, and avoids race conditions where data changes between page fetches.
