Here's a summary of all changes:

**Orders — fix `deleted_at` not being set:**
- `src/services/order-service.js`: `softDeleteOrder` now sets `deleted_at: db.fn.now()` alongside the existing `status: 'deleted'`. This also fixes the bug in `user-service.js` where `getUserOrders` filters on `whereNull('deleted_at')` — previously that filter was silently broken.

**Products — align with `deleted_at` pattern:**
- `src/db.js`: Added `deleted_at` nullable timestamp to the products table schema.
- `src/services/product-service.js`: Added `whereNull('deleted_at')` to `listProducts()` and a new `softDeleteProduct()` function that sets both `is_active: false` and `deleted_at`.
- `src/routes/products.js`: Added `DELETE /:id` endpoint using `softDeleteProduct`, and the `GET /:id` route now also checks `product.deleted_at`.
- `migrations/004-add-deleted-at-to-products.js`: New migration to add the `deleted_at` column to existing databases.
