**PATCH /api/orders/:id** — allows updating `product_name`, `quantity`, `total_cents`, and `product_id` on orders that are still `pending` or `confirmed`.

Guards:
- 404 if order doesn't exist or is deleted
- 400 if order is `shipped`, `delivered`, or `cancelled` (can't modify after shipping)
- 400 if no valid fields provided
- Only whitelisted fields are accepted — ignores anything else in the body (e.g. `status`, `user_id`)
