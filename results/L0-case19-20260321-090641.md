Here's a summary of all changes:

**Orders (`softDeleteOrder` fix + query alignment):**
- `softDeleteOrder` now sets `deleted_at` instead of `status: 'deleted'`
- All order queries (`listOrders`, `searchOrders`, stats) switched from `.where('status', '!=', 'deleted')` to `.whereNull('deleted_at')`
- Route guards changed from `order.status === 'deleted'` to `order.deleted_at`
- This also fixes a latent bug: `getUserOrders` in user-service was already filtering on `whereNull('deleted_at')`, but `deleted_at` was never being set — so soft-deleted orders were still showing up for users

**Products (aligned to same pattern):**
- Added `deleted_at` column to products schema in `db.js`
- `listProducts` now uses `.whereNull('deleted_at')` instead of `.where('is_active', true)`
- Product detail route checks `product.deleted_at` instead of `!product.is_active`
- Added `softDeleteProduct` function in product-service

One thing to note: the `is_active` column is still in the schema. If it's used elsewhere or has a different semantic meaning (e.g., temporarily hiding a product vs. permanently deleting it), you may want to keep it. Otherwise it can be removed in a follow-up.
