Fixed. Both the `GET /:id` and `DELETE /:id` routes now validate the order ID using the already-imported `validateId` utility. 

- `GET /api/orders/abc` → **400** `{"error": "Invalid order ID"}` (was 500)
- `DELETE /api/orders/abc` → **400** `{"error": "Invalid order ID"}` (was 500)

The `validateId` function rejects non-numeric, negative, and zero IDs by parsing to an integer and checking it round-trips correctly.
