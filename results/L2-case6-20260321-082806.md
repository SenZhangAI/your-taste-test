**Root cause:** `getOrder` had no `deleted` filter despite its JSDoc promising one. The route layer had a compensating check, but the bug was in the service contract itself — any other caller of `getOrder` (including `createOrder` which returns the newly created order via `getOrder(id)`) would leak deleted records.

**What changed:**
- `order-service.js:48` — added `where('status', '!=', 'deleted')` to match the documented contract
- `routes/orders.js` — removed redundant `order.status === 'deleted'` checks in GET `/:id` and DELETE `/:id` since the service now enforces this
