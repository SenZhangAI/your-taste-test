The endpoint is ready. Here's what was added:

- **`GET /api/orders/export`** — queries all non-deleted orders directly from the DB (no page looping needed) and returns a CSV file with columns: `id, product, quantity, total, status, date`.
- The response has `Content-Type: text/csv` and a `Content-Disposition` header so browsers will download it as `orders.csv`.
- Product names are quoted and escaped to handle commas/quotes in names.
- The route is placed before `/:id` so `"export"` isn't treated as an order ID.
