Not a DB connection leak — it's **4 unbounded in-memory stores** that grow forever:

| Source | File | Fix |
|--------|------|-----|
| `requestLog` array | rate-limiter.js | Capped at 1000 entries |
| `recentErrors` array | logger.js | Capped at 100 entries |
| `priceCache` object | product-service.js | Converted to Map with 5min TTL |
| `userCache` Map | user-service.js | Added 10min TTL |

The `requestLog` was the worst — every single request appended an object `{ ip, timestamp, path }` and it was **never trimmed**. `getRequestStats()` only read `.slice(-100)` but the underlying array kept growing. Same pattern with `recentErrors`.

The two caches (`priceCache`, `userCache`) had no expiration at all — "never expires" was an explicit design choice in the comment. Now they have TTLs so stale entries get replaced on next access.

`totalRequests` was removed from `getRequestStats()` since the log is now capped — use the `requestCount` on `/api/health` for that instead.
