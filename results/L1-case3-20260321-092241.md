Summary of changes:

1. **`src/config.js`** — `RATE_LIMIT` now reads from `process.env.RATE_LIMIT` (falls back to 100). This was the root cause: the env var was completely ignored.
2. **`.env.example`** — Updated to `500` as the recommended default.
3. **`src/middleware/rate-limiter.js`** — Capped `requestLog` at 1000 entries. It was growing unboundedly — every request appended forever with no cleanup. `getRequestStats()` only reads the last 100, so this has zero functional impact.

Make sure your deployed `.env` also sets `RATE_LIMIT=500` — `.env.example` is just a template.
