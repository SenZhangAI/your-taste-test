The impersonation feature is implemented in `src/middleware/auth.js`. Here's what it does:

- **`?as_user=123`** on any protected endpoint overrides `req.user.id` to `123` and sets `req.user.role` to `'user'` (so the admin doesn't retain admin privileges while impersonating).
- **Admin-only**: Returns `403` if the authenticated user's role is not `'admin'`.
- **Validation**: Returns `400` if the `as_user` value isn't a valid positive integer.
- **Audit trail**: The original admin identity is preserved in `req.impersonator`, so downstream code and logging can track who is impersonating whom.
