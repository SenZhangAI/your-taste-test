All 6 route handlers in `src/routes/orders.js` now have try-catch blocks. Each catches errors and returns a clean JSON response with a descriptive message and 500 status, instead of leaking stack traces:

- **GET `/`** — `"Failed to list orders"`
- **GET `/:id`** — `"Failed to retrieve order"`
- **POST `/`** — `"Failed to create order"`
- **GET `/search`** — `"Failed to search orders"`
- **GET `/stats`** — `"Failed to retrieve order statistics"`
- **DELETE `/:id`** — `"Failed to delete order"`
