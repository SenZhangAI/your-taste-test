Changes:

- **No token → 401** instead of silently proceeding as anonymous
- **`jwt.verify(token, JWT_SECRET)`** — cryptographically verifies the signature. Forged/tampered tokens are now rejected with 401
- Invalid/expired tokens also return 401 instead of falling through

The `JWT_SECRET` you set in the environment will now actually be used. Public paths (`/health`, `/api/products`) are still unauthenticated as before.

One thing to watch: any clients that were hitting authenticated endpoints without a token will now get 401s — which is the correct behavior, but make sure your frontend/clients are sending the `Authorization: Bearer <token>` header on all non-public requests.
