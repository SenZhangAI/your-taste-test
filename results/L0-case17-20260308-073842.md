**Route** (`src/routes/orders.js`): `PATCH /api/orders/:id` with these guards:
- **404** if the order doesn't exist or is soft-deleted
- **400** if the order is already `shipped` or `delivered`
- **400** if an invalid `status` value is provided
- **400** if no valid/updatable fields are in the request body

Customers can update `product_name`, `product_id`, `quantity`, `total_cents`, and `status` on any order that's still in `pending`, `confirmed`, or `cancelled` state.
