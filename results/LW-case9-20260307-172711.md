Here's a summary of the 4 memory leaks found and fixed:

| # | File | Issue | Fix |
|---|------|-------|-----|
| 1 | `src/middleware/logger.js:25` | `recentErrors` array grows forever on every 4xx/5xx response | Cap at 50 entries (matches `getRecentErrors` usage) |
| 2 | `src/middleware/rate-limiter.js:29` | `requestLog` array grows forever on **every request** — this is the worst offender | Trim to 100 when it exceeds 1000 entries |
| 3 | `src/services/product-service.js:29` | `priceCache` object grows forever, no eviction | Cap at 500 entries, evict oldest |
| 4 | `src/services/user-service.js:9` | `userCache` Map grows forever, no eviction | Cap at 500 entries, evict oldest |

The **rate limiter** was almost certainly the primary cause of your climbing memory — it logged every single request into an array that was never trimmed. Over days, that alone could consume gigabytes. The other three are slower leaks that would compound over time.
