The problem: `getProductPrice()` has a module-level `priceCache` that caches forever. Your PATCH endpoint updates the DB and reads back via `getProduct()` (no cache, always fresh), so the response looks correct. But `getProductPrice()` still returns the stale cached value for any subsequent order validation.

**Changes:**
- `product-service.js` — added `invalidatePriceCache(productId)` to delete the cached entry
- `products.js` — call `invalidatePriceCache()` right after the DB update in the PATCH handler

**Broader observation:** The split between `getProduct()` (always fresh) and `getProductPrice()` (silently cached) is a consistency trap. If this cache isn't pulling its weight in production (i.e., you're not doing high-volume batch price lookups), consider removing `priceCache` entirely and just using `getProduct().price_cents`. One code path for product data = no cache-coherency bugs.
