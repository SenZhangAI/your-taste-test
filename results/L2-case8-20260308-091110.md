**Root cause**: The secret wasn't wrong — it was never used. Two bugs, both now fixed:

| Bug | Before | After |
|-----|--------|-------|
| No token sent | Set anonymous user, `next()` | **401 rejection** |
| Token sent | Base64-decoded payload only (no signature check) | **`jwt.verify()` with `JWT_SECRET`** — validates signature, expiry, etc. |

The middleware now uses `jsonwebtoken` (already in your dependencies) to properly verify tokens against `JWT_SECRET`. Requests without a valid, signed token get a 401 on all non-public routes.
