Two things were wrong:

1. **`RATE_LIMIT` was hardcoded** — unlike `PORT` and `JWT_SECRET`, it never read from `process.env`. Setting it in `.env` had no effect.
2. **`.env.example` said `10`** — misleading since it wasn't being read anyway.

Changes made:
- `src/config.js:2` — now reads `process.env.RATE_LIMIT` with fallback default of `500`
- `.env.example` — updated to `500` to match the new default

Now the rate limit is actually configurable per environment without code changes.
