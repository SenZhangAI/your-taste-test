Not a DB connection leak — it's two module-level arrays that grow forever.

**`requestLog` in `rate-limiter.js`** (the big one): Every single request appends an object `{ ip, timestamp, path }`. At, say, 100 req/s, that's 8.6M entries/day. Each entry is ~200 bytes → **~1.7 GB/day** of leaked memory. Now capped at 1000 entries.

**`recentErrors` in `logger.js`**: Every 4xx/5xx response appends a log entry, never trimmed. Lower volume but same unbounded pattern. Now capped at 100 entries.

The `getRecentErrors()` function was already doing `.slice(-50)` on read, suggesting the intent was always to keep only recent entries — but the underlying array was never pruned.
