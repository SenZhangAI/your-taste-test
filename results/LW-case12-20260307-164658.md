Fixed. The `validateId` function was already imported but unused. Now both `GET /api/orders/:id` and `DELETE /api/orders/:id` validate the ID parameter first:

- **Invalid ID** (e.g., `abc`, `-1`, `0`) → **400** with `{"error": "Invalid order ID"}`
- **Valid ID but not found** → **404** with `{"error": "Order not found"}`

This prevents the 500 error caused by passing a non-numeric string like `abc` directly to the database query.
