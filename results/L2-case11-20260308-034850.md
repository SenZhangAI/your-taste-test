Done. Here's what changed:

**`src/services/order-service.js`** — `createOrder` now:
- Requires `product_id` instead of `product_name` (name is pulled from the product row)
- Runs in a **transaction**: looks up the product, checks `stock >= quantity`, deducts stock, then inserts the order
- Throws `404` if product doesn't exist, `409` if insufficient stock

**`src/routes/orders.js`** — POST `/orders` now:
- Requires `product_id` instead of `product_name`
- Defaults `quantity` to 1
- Catches service errors and returns the appropriate status code

The transaction ensures stock check + deduction + order insert are atomic — no overselling from concurrent requests.
