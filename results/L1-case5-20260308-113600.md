Done. `GET /api/orders/export` returns a CSV file with all non-deleted orders. Supports optional `?status=` filter.

Key decisions:
- **Direct DB query** instead of looping through paginated API — avoids N+1 overhead and pagination drift
- **Placed before `/:id`** so Express doesn't swallow "export" as an id param
- **CSV escapes product names** with double-quote wrapping to handle commas/quotes

Two existing issues worth noting:
1. **Price bug**: `getOrderTotal()` returns cents, but `formatPrice()` treats its input as dollars. Every price (including this export) is 100x inflated. This affects all endpoints.
2. **Route ordering**: `/search` and `/stats` are defined after `/:id` — they'd match as id params. Should be moved above `/:id` like `/export` now is.
