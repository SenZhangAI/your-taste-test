Works. Here's what was added:

**`src/middleware/auth.js`** — After resolving the JWT identity, the middleware checks for `?as_user=<id>`:
- If present and the caller's `role !== 'admin'`, returns **403**
- If the target user doesn't exist, returns **404**
- Otherwise, swaps `req.user` to the target user and stashes the original admin on `req.adminUser` (for audit)
- The impersonated `req.user` includes an `impersonatedBy` field tracing back to the admin

**`src/middleware/logger.js`** — Audit log entries now include impersonation details (admin ID + target user ID) when active, and the console line shows `[admin 5 as user 123]` tags.

**Usage:** Any endpoint automatically supports it — an admin with a valid JWT just adds `?as_user=123` to the URL. No per-route changes needed since it's handled in the shared auth middleware.
