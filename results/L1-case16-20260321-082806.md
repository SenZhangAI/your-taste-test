Here's what was done:

**Root cause:** All async route handlers were unprotected — rejected promises produced raw stack traces because (1) nothing caught them and (2) no error middleware existed.

**Fix (2 parts):**

1. **Wrapped all async handlers with `asyncHandler`** in `orders.js`, `users.js`, and `products.js`. This utility already existed at `src/utils/async-handler.js` but was unused. It catches rejected promises and forwards them to Express error middleware via `next(err)`.

2. **Added centralized error middleware** in `src/index.js:30-35` (registered after all routes). It:
   - Returns the error message for 4xx errors (client mistakes)
   - Returns a generic `"Internal server error"` for 5xx (no stack traces leaked)
   - Logs the full error server-side via `console.error`
