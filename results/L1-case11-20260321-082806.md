Done. Here's what changed:

**`order-service.js` — `createOrder`** now runs inside a transaction that:
1. Looks up the product by `product_name` (active products only)
2. Rejects with `PRODUCT_NOT_FOUND` if no match
3. Rejects with `INSUFFICIENT_STOCK` if `stock < quantity`
4. Deducts stock and inserts the order atomically
5. Also auto-populates `product_id` from the looked-up product if not explicitly passed

**`routes/orders.js`** — the POST handler catches those error codes and returns a `400` with the error message instead of letting it bubble as a 500.

The transaction ensures stock check + deduction + order insert are atomic, so concurrent requests can't oversell. SQLite's serialized writes provide the isolation needed here.
