**`PATCH /api/orders/:id`** — updates an order before it ships.

**Route** (`src/routes/orders.js:80-103`):
- Validates the `:id` param with `validateId`
- Validates `status` against `ORDER_STATUSES`, `quantity` and `total_cents` as positive integers
- Returns the updated order with formatted total/date

**Service** (`src/services/order-service.js:73-95`):
- Blocks updates on orders with status `shipped`, `delivered`, or `deleted` (returns 409)
- Whitelists updatable fields: `product_name`, `product_id`, `quantity`, `total_cents`, `status`, `category`
- Returns 400 if no valid fields are provided
- Sets `updated_at` automatically

**Response codes:**
- `200` — order updated successfully
- `400` — invalid ID, invalid field values, or no valid fields
- `404` — order not found
- `409` — order already shipped/delivered/deleted
