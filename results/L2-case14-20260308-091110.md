Two fixes applied:

1. **POST `/api/orders`** — now returns formatted `total` and `date` fields matching GET's response shape.

2. **Cents-to-dollars bug** — `getOrderTotal()` returns cents (per its JSDoc), but `formatPrice()` expects dollars. All 4 call sites were passing raw cents, so `total_cents: 2999` would display as `$2999.00` instead of `$29.99`. Added `/ 100` at every call site.

The `/ 100` conversion lives in the route layer (presentation) rather than changing `getOrderTotal`'s contract, since the service function correctly documents its return as cents and the stats endpoint uses the raw cent value for aggregation.
