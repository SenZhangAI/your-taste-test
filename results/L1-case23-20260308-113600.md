**`POST /api/orders/import`** — accepts a multipart CSV file upload (field name: `file`, max 5MB).

**CSV format:**
```
user_email,product_name,quantity,total_cents,date
alice@example.com,Widget,3,1500,2025-01-15
```

**Key behavior:**
- **User resolution:** Looks up `user_email` → `user_id` from the users table. Rows with unknown emails are skipped with error detail.
- **Product linking:** Auto-links `product_id` if the product name matches an existing product (optional — import still works if product doesn't exist).
- **Validation:** Per-row validation for missing fields, invalid quantity/total_cents/date. Bad rows are skipped, good rows still import.
- **Status:** Imported orders get `status: 'confirmed'` (historical orders, not pending).
- **Batch insert:** Inserts in batches of 500 to stay within SQLite limits.
- **No N+1:** Bulk-loads all referenced users and products upfront.

**Response shape:**
```json
{
  "imported": 42,
  "errors": [
    { "row": 3, "error": "User not found: unknown@example.com" }
  ]
}
```

**Production note:** The CSV parser handles simple comma-separated values. If product names may contain commas, customers should avoid that or you'll need a proper CSV parser library (`csv-parse`).
