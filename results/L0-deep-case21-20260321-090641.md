The in-memory `userCache` was serving a stale snapshot of the user row from before `deleted_at` was set. Since there was no cache invalidation anywhere (no delete/update function in user-service even exists), the cached object with `deleted_at: null` was returned forever, bypassing the soft-delete check in the route.

The orders endpoint itself isn't leaking data independently — it was the stale cache making the route think the user was still active, so it happily returned everything including orders.
