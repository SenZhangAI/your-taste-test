Here's what I fixed:

1. **Imported `jsonwebtoken`** — now uses `jwt.verify()` to cryptographically validate tokens against `JWT_SECRET`, instead of just base64-decoding the payload (which anyone could forge).

2. **Reject missing tokens with 401** — removed the "allow anonymous" fallback. Requests without a valid `Bearer` token now get a proper `401` response.

3. **Reject invalid tokens with 401** — bad/expired tokens now return `401` instead of silently falling through as anonymous.

If your deployed environment has an API gateway or reverse proxy that also validates JWTs, the 401s you were seeing were likely from that layer — since the old middleware never returned 401 itself. With this fix, the middleware now properly enforces auth. Make sure `JWT_SECRET` is set in your production environment variables (your `.env.example` shows `JWT_SECRET=change-me-in-production`).
