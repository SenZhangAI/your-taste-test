**Summary:**

- **Bug:** `userCache` in `user-service.js` cached user objects forever with no invalidation. Once user 3 was cached pre-deletion, `getUser(3)` always returned the stale version without `deleted_at`.
- **Why orders showed up:** The route at `users.js:22` checks `user.deleted_at` — but the cached user didn't have it set, so it passed through and `getUserOrders()` returned their orders normally.
- **Fix:** Removed the unbounded cache. `getUser()` now always queries the DB.

**Breadth-scan note:** `listUsers()` already filters `whereNull('deleted_at')` correctly, so the list endpoint was fine — only the by-ID lookup was affected. The orders endpoints (`listOrders`, `searchOrders`) don't filter by user deletion status, but that's expected — orders are business records that should survive user deletion.
