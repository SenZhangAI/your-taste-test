Everything looks good. Here's what was added:

### Changes

**Migration** (`migrations/004-add-refund-to-orders.js`):
- `refund_cents` (integer, nullable) — tracks partial or full refund amount
- `cancelled_at` (timestamp, nullable) — when the cancellation happened

**Service** (`src/services/order-service.js`):
- `cancelOrder(id, refundCents)` — validates the order can be cancelled (rejects if already cancelled, delivered, or deleted), ensures refund doesn't exceed order total, then updates status + refund fields

**Route** (`src/routes/orders.js`):
- `POST /api/orders/:id/cancel` — accepts `{ refund_cents }` in body, returns the cancelled order with formatted `refund` field
- `GET /api/orders/:id` — now includes `refund` and `cancelled_at` when the order is cancelled

### API usage

```bash
# Cancel with partial refund (e.g. $5.00 refund on a larger order)
POST /api/orders/42/cancel
{ "refund_cents": 500 }

# Full refund
POST /api/orders/42/cancel
{ "refund_cents": 2999 }
```

Response includes `refund: "$5.00"` and `cancelled_at` timestamp alongside the existing order fields.
