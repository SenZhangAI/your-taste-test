CSV parser works correctly. Here's a summary of what was added:

**New file:** `src/utils/csv.js` — Parses CSV text, validates required headers (`user_email, product_name, quantity, total_cents, date`), returns typed row objects.

**Modified `src/routes/orders.js`:**
- `POST /api/orders/import` — accepts a multipart `file` field (5MB limit) via multer memory storage, parses the CSV, calls the service, returns `{ inserted, failed, errors }`.
- Returns 400 for missing file / bad CSV structure, 422 if all rows fail validation, 200 otherwise.

**Modified `src/services/order-service.js`:**
- `importOrders(rows)` — batch-lookups users by email and products by name, validates each row (quantity > 0, total_cents > 0, valid date, user exists), inserts valid rows in a transaction (batches of 500), returns per-row errors for invalid rows.
- Imported orders get status `confirmed` and `created_at` set to the CSV date.
- `product_id` is resolved from product name when a match exists, `null` otherwise.

**Example usage:**
```
curl -X POST http://localhost:3000/api/orders/import \
  -F file=@orders.csv
```

Where `orders.csv` contains:
```
user_email,product_name,quantity,total_cents,date
alice@example.com,Widget Pro,2,5998,2024-01-15
bob@example.com,Gadget Mini,1,1499,2024-02-20
```
