**Summary of the fix:**

- **Root cause:** `getOrder()` in `src/services/order-service.js:28` queried by `id` only, ignoring the `deleted` status — despite the JSDoc explicitly claiming it filters deleted records.
- **Fix:** Added `.whereNot({ status: 'deleted' })` to the query so deleted orders are filtered at the service layer, protecting all consumers.
- **Cleanup:** Removed the now-redundant `order.status === 'deleted'` checks in both route handlers (`GET /:id` and `DELETE /:id`), since `getOrder` now returns `null` for deleted orders.
