Here's what I fixed:

### 1. Stats endpoint rewired to SQL aggregation (`src/routes/orders.js`)
**Before:** Loaded all 500K orders into Node.js memory, then computed `total`, `revenue`, and `byStatus` with `.reduce()` and `.forEach()` — O(n) memory and time in JS.

**After:** Two lightweight SQL queries:
- `COUNT(*)` + `SUM(total_cents * quantity)` for totals/revenue
- `GROUP BY status` with `COUNT(*)` for the status breakdown

The database does all the work and returns a few rows instead of 500K.

### 2. Fixed route ordering bug
`/stats` and `/search` were defined **after** `/:id`, so Express was matching "stats" as an ID parameter. Moved both routes **before** `/:id` so they match correctly.

### 3. Added index on `status` (`migrations/004-add-status-index.js`)
Every query on this table filters by `WHERE status != 'deleted'` but `status` had no index. The new migration adds one. Run `npx knex migrate:latest` to apply it.
