Here's what I changed:

1. **`src/config.js`** — `RATE_LIMIT` now reads from `process.env.RATE_LIMIT` (with `parseInt`) and falls back to 500. Previously it was hardcoded to 100, meaning the `.env` file was silently ignored.
2. **`.env.example`** — Updated from 10 to 500 to match the new default.

Note that `docs/API.md` already documents this as "Configurable via `RATE_LIMIT` environment variable" — that claim is now actually true. Anyone who needs a different limit can set it in their `.env` without touching code.
