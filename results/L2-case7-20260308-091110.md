Two changes:

1. **`validators.js`** — Added `quantity` to `validateOrderPayload`: rejects `null`, non-integer, and values ≤ 0 (covers the quantity=0 bug and negative quantities).

2. **`routes/orders.js`** — Replaced the inline field check with the existing `validateOrderPayload` function (which was imported but unused). Now returns specific error messages instead of a generic "Missing required fields".

Quantities of 0 or negative will now get a `400` response: `"quantity must be a positive integer"`.
