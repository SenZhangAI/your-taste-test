Here's what was added:

**In `order-service.js`** — `createOrder` now:
1. Looks up the product by `product_name` in the products table
2. Returns a 400 error if the product isn't found
3. Returns a 409 error if `product.stock < requested quantity`
4. Decrements stock by the order quantity on success
5. Also auto-populates `product_id` from the looked-up product if not provided

**In `routes/orders.js`** — The POST handler now catches errors with a `status` property and returns the appropriate HTTP response.
