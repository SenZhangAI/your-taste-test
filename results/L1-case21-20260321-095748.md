**One more thing worth noting**: `getUserOrders` filters by `whereNull('deleted_at')` on the *orders* table, but orders use `status: 'deleted'` for soft-deletion (see `order-service.js:72`), not `deleted_at`. So that filter is a no-op — it never actually excludes deleted orders. That's a separate bug (orders soft-delete via `status`, not `deleted_at`), but it's not what's causing your current issue.

**Summary of the fix:**

1. **Root cause**: `getUser()` had an in-memory cache (`Map`) with no invalidation and no TTL. Once user 3 was cached pre-deletion, the stale copy was served forever, bypassing the `deleted_at` check in the route.
2. **Fix**: Removed the broken cache and added `whereNull('deleted_at')` directly to the query so deleted users are never returned from the service layer.
