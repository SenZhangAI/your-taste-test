Here's what was leaking and what I fixed:

| Source | Problem | Fix |
|--------|---------|-----|
| **`rate-limiter.js` — `requestLog`** | Every request pushed to array, never trimmed | Capped at 1,000 entries, oldest spliced off |
| **`logger.js` — `recentErrors`** | Every error response pushed, never trimmed | Capped at 100 entries |
| **`user-service.js` — `userCache`** | Cached every user forever, no eviction | Added 5-min TTL + max 500 entries |
| **`product-service.js` — `priceCache`** | Cached every price forever as plain object | Switched to Map with 5-min TTL |

The `requestLog` was almost certainly the biggest offender — it stored an object for literally every request with no cleanup at all. Over days of traffic that would eat memory fast. The caches are slower leaks but would also grow unbounded with enough unique users/products.
