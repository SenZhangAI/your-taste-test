**`POST /api/orders/import`** — accepts a CSV file upload via multipart form field `file`.

**Route** (`src/routes/orders.js:32-67`):
- Uses multer with memory storage (5MB limit)
- Validates file presence, header row, and expected columns
- Parses CSV rows and delegates to `importOrders`
- Returns 400 if all rows fail, 200 otherwise

**Service** (`src/services/order-service.js:77-135`):
- Per-row validation: required fields, positive quantity/total_cents, valid date
- Looks up `user_id` from `users` table by email
- Optionally resolves `product_id` from `products` table by name (null if not found)
- Inserts all valid rows in a single transaction with status `confirmed`
- Returns `{ imported: N, errors: [...] }` with per-row error details

**Example usage:**
```bash
curl -X POST http://localhost:3000/api/orders/import \
  -H "Authorization: Bearer <token>" \
  -F "file=@orders.csv"
```

Where `orders.csv` looks like:
```
user_email,product_name,quantity,total_cents,date
alice@example.com,Widget,2,5000,2024-01-15
bob@example.com,Gadget,1,2999,2024-02-20
```
