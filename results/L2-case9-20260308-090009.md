## Summary

**Not a DB connection leak.** The memory growth comes from two module-level arrays that accumulate entries on every request without any cap:

| Source | Growth rate | Fix |
|--------|-----------|-----|
| `rate-limiter.js` `requestLog` | Every request | Capped at 1000 entries, trims when exceeding 2x |
| `logger.js` `recentErrors` | Every 4xx/5xx | Capped at 100 entries, same trim strategy |
| `orders.js` `/stats` | Per-request spike | Replaced with aggregate SQL query |

The `splice` approach (trim at 2x, keep last N) avoids shifting on every push while keeping memory bounded. After these changes the process memory should stabilize and you won't need periodic restarts.
