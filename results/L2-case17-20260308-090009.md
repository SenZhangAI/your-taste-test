**PATCH /api/orders/:id** — updates an order's `product_name`, `quantity`, `total_cents`, or `product_id`.

Key guardrails:
- **Pre-shipment only**: Updates blocked once status moves past `pending`/`confirmed` (returns 409)
- **Whitelist approach**: Only the four updatable fields are accepted; everything else is ignored
- **Validates**: order existence (404), valid ID format (400), at least one valid field provided (400)
- Automatically sets `updated_at` on every update
