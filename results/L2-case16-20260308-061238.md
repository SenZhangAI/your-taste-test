Here's what I did instead of per-route try-catch:

1. **`src/utils/async-handler.js`** — one-line wrapper that catches async rejections and forwards them to `next(err)`
2. **`src/index.js`** — added centralized error middleware that returns `{ error: "..." }` JSON with proper status codes (never stack traces)
3. **All three route files** wrapped with `asyncHandler()` — orders, products, and users all had the same vulnerability

This is the standard Express 4 pattern. Any unhandled error in any async route now returns clean JSON like `{"error": "Internal server error"}` instead of a raw stack trace.
