Here's what was happening:

**Root cause:** `product-service.js` has a module-level `priceCache` object (line 7) that caches product prices indefinitely. `getProductPrice()` populates it on first lookup but **never invalidates it**. The `PATCH /products/:id` route updates the DB price directly but doesn't touch the cache, so subsequent order validation via `getProductPrice()` returns the stale cached value.

**Fix:**
1. Added `clearPriceCache(productId)` to `product-service.js` to delete a specific cache entry
2. Called it in the `PATCH /products/:id` route right after the DB update, so the next `getProductPrice()` call will fetch the fresh value from the DB
