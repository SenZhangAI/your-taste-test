Everything looks good. Here's what was added:

### Changes

**New migration** `migrations/004-add-cancellation-fields.js` — adds `refund_cents` (nullable int) and `cancelled_at` (nullable timestamp) to orders table.

**`src/db.js`** — same columns added to `initDB` for fresh databases.

**`src/services/order-service.js`** — new `cancelOrder(id, refundCents)` function that sets status to `cancelled`, stores the refund amount and timestamp.

**`src/routes/orders.js`**:
- **`POST /:id/cancel`** — cancellation endpoint. Accepts optional `refund_cents` in body (defaults to full refund). Validates: order exists, not already cancelled, refund doesn't exceed order total, non-negative number.
- **`GET /:id`** — now includes a `refund` block when order is cancelled, showing `amount`, `refund_cents`, `is_partial`, and `cancelled_at`.

### API usage

```bash
# Full refund (default)
POST /api/orders/123/cancel
{}

# Partial refund
POST /api/orders/123/cancel
{ "refund_cents": 500 }
```

Response includes:
```json
{
  "refund": {
    "amount": "$5.00",
    "refund_cents": 500,
    "is_partial": true,
    "cancelled_at": "2026-03-21"
  }
}
```
