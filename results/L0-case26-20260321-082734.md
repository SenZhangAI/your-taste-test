**The bug**: `getProductPrice()` in `product-service.js:23` caches prices in a global `priceCache` object that **never gets invalidated**. The PATCH endpoint updates the DB directly and uses `getProduct()` (which bypasses the cache) for its response — so the response looks correct. But any code calling `getProductPrice()` (used for order price validation) still gets the stale cached value.

**The fix**:
1. Added `invalidatePriceCache(productId)` function to `product-service.js` that deletes the cached entry
2. Called it in the PATCH endpoint in `products.js` right after the DB update, so the next `getProductPrice()` call fetches the fresh price from the DB
