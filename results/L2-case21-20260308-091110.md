**Summary:**

The problem wasn't the orders endpoint — it was a **stale in-memory cache** in `getUser()`. The cache stored user objects forever ("users rarely change"), so after you set `deleted_at` in the DB, the cached version without `deleted_at` kept being served.

Changes:
- **Removed the `userCache`** — an unbounded, never-invalidating cache on mutable data is a bug waiting to happen
- **Added `whereNull('deleted_at')` to the `getUser` query** — consistent with how `listUsers` already works, filtering at the DB level instead of relying on application-level checks
- **Simplified the route check** — no need for the redundant `user.deleted_at` check since the query handles it
