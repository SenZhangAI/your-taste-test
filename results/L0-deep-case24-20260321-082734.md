Here's a summary of the changes:

**New migration** — `migrations/004-add-cancellation-fields.js`: Adds `refund_cents` (nullable int) and `cancelled_at` (nullable timestamp) to the orders table.

**`src/db.js`** — Added the same two columns to the inline schema initialization fallback.

**`src/services/order-service.js`** — Added `cancelOrder(id, refundCents)` which sets status to `cancelled`, records the refund amount and cancellation timestamp, and returns the updated order.

**`src/routes/orders.js`** — Two changes:
- **`POST /orders/:id/cancel`** — New endpoint that accepts `{ refund_cents }` in the body. Validates the order exists, isn't already cancelled/deleted, and that `refund_cents` is a non-negative integer not exceeding the order total. Returns the cancelled order with formatted `total`, `refund`, and `cancelled_at`.
- **`GET /orders/:id`** — Now includes `refund` and `cancelled_at` fields in the response when the order status is `cancelled`.
