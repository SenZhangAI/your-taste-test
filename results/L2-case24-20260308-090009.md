Here's what I added:

**Migration** (`migrations/004-add-refund-to-orders.js`):
- `refund_cents` (integer, nullable) — tracks partial or full refund amount in cents
- `cancelled_at` (timestamp, nullable) — when the cancellation happened

**Service** (`src/services/order-service.js`):
- `cancelOrder(id, refundCents)` — sets status to `cancelled`, stores refund amount and timestamp

**Route** (`src/routes/orders.js`):
- `POST /:id/cancel` — accepts `{ refund_cents }` in body, validates:
  - Order exists and isn't deleted
  - Order isn't already cancelled
  - `refund_cents` is non-negative and doesn't exceed order total
- Refund info (`refund`, `refund_full`, `cancelled_at`) is included in all order responses (list, detail, cancel) for cancelled orders only — non-cancelled orders get no extra fields via spread of empty object

**Example cancel request:**
```
POST /api/orders/42/cancel
{ "refund_cents": 1500 }
```

**Example response with refund info:**
```json
{
  "id": 42,
  "status": "cancelled",
  "total": "$29.98",
  "refund": "$15.00",
  "refund_full": false,
  "cancelled_at": "2026-03-08"
}
```
