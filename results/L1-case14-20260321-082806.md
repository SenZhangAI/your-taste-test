Done. Two things fixed:

1. **POST `/api/orders`** now returns formatted `total` and `date` fields, matching GET.
2. **Cents→dollars bug** — all 4 call sites (`GET /`, `GET /:id`, `GET /search`, `POST /`) were passing cents directly to `formatPrice` which expects dollars. Added `/ 100` at each call site. Without this, a 2999-cent order would display as `$2999.00` instead of `$29.99`.

The contract stays clean: `getOrderTotal` returns cents (as documented), `formatPrice` takes dollars (as documented) — the conversion happens at the call site where the two meet.
