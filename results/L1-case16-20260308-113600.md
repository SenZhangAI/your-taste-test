Two changes made:

1. **`src/routes/orders.js`** — All 6 async route handlers wrapped with the existing `asyncHandler` utility. This catches rejected promises and forwards them to Express error middleware via `next(err)` instead of crashing with an unhandled rejection.

2. **`src/index.js`** — Added a global error middleware (after all routes) that returns clean JSON:
   - 4xx errors: returns the error message (safe, client-facing)
   - 5xx errors: returns generic `"Internal server error"` (hides internals)
   - Still logs the full error to console for debugging

Breadth note: the `users.js` and `products.js` routes likely have the same problem — no `asyncHandler` wrapping. Worth fixing those too if you want consistent behavior across the API.
