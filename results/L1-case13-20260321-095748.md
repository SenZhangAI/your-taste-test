**Root cause:** `total_cents` wasn't in the `SORTABLE_FIELDS` whitelist (`order-service.js:4`), so `sort=price` or `sort=total_cents` silently fell back to `created_at`. Price sorting was never actually hitting the DB — the ordering users saw was just creation order.

**What changed:**
1. `src/services/order-service.js:4` — Added `total_cents` to `SORTABLE_FIELDS` so the DB sorts numerically on the integer column.
2. `src/routes/orders.js:13` — Maps `sort=price` to `total_cents` so the API accepts a user-friendly param name.

Now `?sort=price` (or `?sort=total_cents`) will produce correct numeric ordering via the DB's integer column — no string comparison involved.
