Here's what I found and fixed:

**The main culprits (HIGH):**
- **`requestLog`** in `rate-limiter.js` — every request appended to an array that **never got trimmed**. At steady traffic this grows without bound. Now capped at 10K entries, halved when it hits the limit.
- **`recentErrors`** in `logger.js` — same pattern, every 4xx/5xx error appended forever. `getRecentErrors()` only sliced for display but never trimmed the source array. Now capped at 500.

**Contributing (MEDIUM):**
- **`priceCache`** in `product-service.js` — plain object cache with no eviction. Now evicts oldest half when it hits 1000 entries.
- **`userCache`** in `user-service.js` — Map cache marked "never expires". Now evicts oldest entry at 1000 entries.

The `requestLog` array is almost certainly your primary leak — it grows linearly with every single request and each entry holds strings (ip, path, timestamp). A few days of traffic would easily accumulate millions of entries.
