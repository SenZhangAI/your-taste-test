Changes:
1. **Missing token → 401** instead of silently passing through as anonymous
2. **`jwt.verify()`** now actually validates the signature against `JWT_SECRET`, replacing the raw base64 decode that accepted any forged token
3. **Invalid/expired token → 401** instead of falling through to anonymous

The `JWT_SECRET` env var you already have set will now actually be used. Public paths (`/health`, `/api/products`) remain unauthenticated as intended.
