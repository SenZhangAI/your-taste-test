Fixed. The problem was purely route ordering тАФ in Express, `/:id` is a catch-all parameterized route that matched `/search` before the literal `/search` route could. Now the route order is:

1. `GET /` тАФ list
2. `GET /search` тАФ search  
3. `GET /stats` тАФ statistics
4. `GET /:id` тАФ single order (after all literal paths)
5. `POST /` тАФ create
6. `DELETE /:id` тАФ soft delete
