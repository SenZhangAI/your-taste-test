Good news: it's not a database connection leak. Bad news: there are **4 unbounded in-memory data structures** that grow forever. Here's what's eating your memory:

### 1. `recentErrors` array — `src/middleware/logger.js:6`
Every 4xx/5xx response pushes to this array and it's **never trimmed**. `getRecentErrors()` slices the last 50 but the underlying array keeps growing.

### 2. `requestLog` array — `src/middleware/rate-limiter.js:4`
**Every single request** gets pushed here. The cleanup interval (line 7-17) only cleans the `hits` Map, not `requestLog`. On a busy API this is your biggest leak.

### 3. `userCache` Map — `src/services/user-service.js:3`
Caches every user lookup forever — no TTL, no eviction, no size limit.

### 4. `priceCache` object — `src/services/product-service.js:7`
Same pattern — caches every product price lookup forever.

Want me to fix all four? The approach would be:
- **logger**: trim `recentErrors` to the last 50 on each push
- **rate-limiter**: trim `requestLog` in the existing cleanup interval
- **userCache/priceCache**: add TTL-based expiration (e.g. 5 minutes)
