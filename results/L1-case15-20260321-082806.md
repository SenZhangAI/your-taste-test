The bug: `config.js` had `RATE_LIMIT` hardcoded to `100` — it never read `process.env.RATE_LIMIT`. Fixed to `parseInt(process.env.RATE_LIMIT, 10) || 100`, so your `.env` value of `10` will now take effect, with `100` as the fallback default.

Breadth-scan: `PORT` and `JWT_SECRET` already read from `process.env` correctly. `PAGE_SIZE` is hardcoded but isn't in `.env.example`, so that's intentional. No other config issues.

One thing to note: make sure you're loading `.env` via `dotenv` (or similar) before `config.js` is imported. If you're not already doing that, `process.env.RATE_LIMIT` won't be populated from the `.env` file.
