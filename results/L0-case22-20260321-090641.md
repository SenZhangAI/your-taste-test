The LIKE query is fine — the issue was **route ordering**. Express evaluates routes top-down, and `/:id` was defined before `/search`. So `/api/orders/search` matched `/:id` with `id = "search"`, which naturally returned no order → 404.

Moved the `/search` route above `/:id` so it matches first. The `/stats` route on line 66 has the same problem — if you ever hit that endpoint, you'd want to move it above `/:id` too.
