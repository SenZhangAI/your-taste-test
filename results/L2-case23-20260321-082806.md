**`POST /api/orders/import`** — accepts a multipart form upload with field name `file`.

**CSV format** (with header row):
```
user_email,product_name,quantity,total_cents,date
alice@example.com,Widget Pro,2,4999,2024-06-15
```

**Key behaviors:**
- Looks up `user_email` → `user_id` via DB (cached per-request to avoid repeated queries for same email)
- Validates each row: required fields, positive integers, valid date, user exists
- Imported orders get `status: 'delivered'` (past orders)
- CSV `date` becomes `created_at`
- Batch inserts (500 rows per batch) for performance
- 5MB file size limit
- Partial success — valid rows import, invalid rows reported with line numbers

**Response:**
```json
{
  "imported": 48,
  "skipped": 2,
  "total": 50,
  "errors": [
    { "line": 12, "error": "User not found: \"unknown@test.com\"" }
  ]
}
```

- `src/routes/orders.js` — new `/import` route with multer
- `src/services/order-service.js` — `importOrders()` for batch insert
- `src/services/user-service.js` — `getUserByEmail()` for email lookup
