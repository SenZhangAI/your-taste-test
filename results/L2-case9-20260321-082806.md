All four leaks are fixed. Here's what changed:

**`src/middleware/rate-limiter.js`** — `requestLog` array now caps at ~200 entries. When it exceeds 400, it splices down to the most recent 200. The `splice` with `2x` threshold avoids splicing on every single request.

**`src/middleware/logger.js`** — Same pattern for `recentErrors`, capped at ~100 entries. The `getRecentErrors()` call only returns 50, so 100 is more than enough buffer.

**`src/services/user-service.js`** — `userCache` now stores entries with timestamps and a 10-minute TTL. A cleanup interval runs every 5 minutes to evict expired entries.

**`src/services/product-service.js`** — Converted `priceCache` from a plain object to a `Map` with the same TTL + eviction pattern. This also fixes a stale-price bug — previously cached prices never reflected updates from `PATCH /products/:id`.

The DB connection itself (better-sqlite3 via Knex) is fine — it's a single synchronous file handle, no pooling issues there. The memory growth was entirely from these four unbounded in-memory data structures.
