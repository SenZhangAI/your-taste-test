Looks good. Here's the summary:

**Not a database connection leak** — you're using SQLite (better-sqlite3), which is in-process with no connection pool.

**The actual leaks are two unbounded in-memory arrays:**

| File | Array | Growth rate | What it stores |
|---|---|---|---|
| `src/middleware/rate-limiter.js:30` | `requestLog` | **Every request** | `{ip, timestamp, path}` objects |
| `src/middleware/logger.js:26` | `recentErrors` | Every 4xx/5xx response | `{method, path, status, duration, timestamp}` objects |

The `requestLog` is the primary culprit — it grows with every single request and is never trimmed. Over days of traffic, this array accumulates hundreds of thousands (or millions) of objects. Both arrays only exist to serve monitoring endpoints that display the last 50-100 entries, but the full history was kept in memory.

**Fix:** Both arrays are now capped — `requestLog` at 1000 entries and `recentErrors` at 100. Old entries are spliced out when the limit is exceeded.
