**`PATCH /api/orders/:id/status`** — update an order's status.

Request body:
```json
{ "status": "confirmed" }
```

Validation:
- ID must be a positive integer
- `status` must be one of the existing `ORDER_STATUSES` (`pending`, `confirmed`, `shipped`, `delivered`, `cancelled`)
- State machine enforces valid transitions — returns 400 with a clear message for invalid ones (e.g., `Cannot transition from 'delivered' to 'pending'`)
- Returns 404 for missing/deleted orders

Response: the full order object with formatted `total` and `date`, same shape as `GET /:id`.

**Breadth-scan:** The soft-delete endpoint (`DELETE /:id`) sets status to `'deleted'` directly, bypassing the state machine — that's correct since `'deleted'` is an internal status not exposed via PATCH. The stats endpoint and list queries already filter by `!= 'deleted'`, so no impact there.
