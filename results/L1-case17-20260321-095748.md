Done. Here's what was added:

**`PATCH /api/orders/:id`** — updates order fields before shipment.

- **Allowed fields**: `product_name`, `quantity`, `total_cents`, `product_id` (whitelist — ignores anything else)
- **Status guard**: Only `pending` and `confirmed` orders can be modified. `shipped`/`delivered`/`cancelled`/`deleted` return `409 Conflict`
- **Validation**: Invalid ID → 400, order not found → 404, no valid fields in body → 400
- **Response**: Returns the updated order in the same shape as `GET /:id`

**Breadth-scan notes:**
- The existing `GET /:id` and `DELETE /:id` routes don't validate the `:id` param with `validateId` (they rely on the DB returning null). Not broken, but inconsistent — the new PATCH route does validate. I left the existing routes as-is since they work correctly and this isn't a breaking issue.
- `validateId` and `ORDER_STATUSES` are imported in the route file but `validateId` wasn't used by existing routes, and `ORDER_STATUSES` isn't used at all. Both were already there before this change.
