Here's what was added:

**Service** (`src/services/order-service.js`): New `updateOrder(id, fields)` function that applies the given fields and sets `updated_at`.

**Route** (`src/routes/orders.js`): New `PATCH /api/orders/:id` endpoint with these rules:
- Returns **404** if the order doesn't exist (or is soft-deleted)
- Returns **409** if the order has already shipped or been delivered
- Only allows updating `product_name`, `product_id`, `quantity`, `total_cents`, and `status` — ignores anything else from the body
- Validates that `status`, if provided, is one of the known `ORDER_STATUSES`
- Returns **400** if no valid fields are provided
- Returns the updated order with formatted total and date on success
