The bug was on line 25-27 in `src/services/order-service.js`. The total count query was building a fresh query against `db('orders')` with only the `!= deleted` filter, ignoring the `status` filter entirely. So when filtering for "pending", you'd get the correct 3 pending orders but the total would reflect all 47 non-deleted orders.

Fixed by reusing `query.clone()` (which already has both the `!= deleted` and the status filter applied) for the count query.
