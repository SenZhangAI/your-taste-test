The `priceCache` at `product-service.js:7` was a module-level object that cached prices permanently with no invalidation. When the PATCH endpoint updated the DB, the cache still held the old value, so `getProductPrice()` returned stale data.

Removed the cache entirely — there's no batch operation in this codebase that would benefit from it, and a single DB lookup per order creation is fine. If batch perf becomes a concern later, the right approach would be a cache with explicit invalidation on price updates, not a write-through-forget cache like this.
