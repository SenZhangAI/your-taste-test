The `users` table has no `role` column — role only comes from the JWT payload. The fallback `|| 'user'` handles this correctly. The implementation is complete.

Here's what was added to `src/middleware/auth.js`:

- **Import** `getUser` from the user service
- **Made `authenticate` async** to support the DB lookup
- **After auth resolves**, checks for `?as_user=<id>` query param
- **Guards**: returns `403` if the requester isn't an admin, `404` if the target user doesn't exist
- **Swaps `req.user`** to the target user's identity (`id` + `role`, defaulting to `'user'`)
- **Preserves the original admin** in `req.impersonatedBy` for audit/logging purposes

Usage: any endpoint with `?as_user=123` will execute as user 123, e.g. `GET /api/orders?as_user=123`. Only requests where the JWT payload contains `role: 'admin'` are allowed to impersonate.
