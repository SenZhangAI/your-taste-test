**`PATCH /api/orders/:id`** — updates an order's `product_name`, `product_id`, `quantity`, and/or `total_cents`.

Guards:
- **404** if order doesn't exist or is soft-deleted
- **409** if order status is beyond `pending`/`confirmed` (already shipped/delivered/cancelled)
- **400** if the request body contains no valid updatable fields

Only whitelisted fields are written to the DB — callers can't modify `id`, `status`, `user_id`, or timestamps directly.
