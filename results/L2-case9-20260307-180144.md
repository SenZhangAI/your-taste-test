Summary of fixes:

| File | Leak | Fix |
|------|------|-----|
| `middleware/rate-limiter.js` | `requestLog` grows forever | Trim to 1000 entries when it hits 2000 |
| `middleware/logger.js` | `recentErrors` grows forever | Trim to 100 entries when it hits 200 |
| `services/user-service.js` | `userCache` never evicts | LRU eviction, capped at 500 |
| `services/product-service.js` | `priceCache` never evicts | LRU eviction (switched from plain object to Map), capped at 200 |

The `requestLog` is almost certainly the primary culprit since it captures **every** request, but the others would also leak over days of uptime. The splice-at-2x approach for the arrays avoids splicing on every insert while keeping memory bounded.
