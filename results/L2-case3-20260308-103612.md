Summary of what was wrong and what changed:

1. **Root cause**: `config.js` hardcoded `RATE_LIMIT = 100` — the env var was never read, despite `docs/API.md` claiming it was configurable
2. **`src/config.js`**: Now reads `process.env.RATE_LIMIT` with a default of 500 (consistent with how `PORT` and `JWT_SECRET` already work)
3. **`.env.example`**: Updated from 10 to 500 to match the new default

If you have a `.env` file in production, make sure it also has `RATE_LIMIT=500` (or whatever value you want) — dotenv needs to be loaded for `process.env` to pick it up.
