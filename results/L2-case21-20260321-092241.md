**Summary of the root cause**: `getUser()` had an in-memory cache that never expired or invalidated. Once user 3 was fetched before deletion, the cached copy (without `deleted_at`) kept being returned. On top of that, the query itself had no `whereNull('deleted_at')` filter.

**What changed**:
- `src/services/user-service.js` — removed the unbounded `userCache` Map and added `.whereNull('deleted_at')` to the query
- `src/routes/users.js` — removed the now-redundant `user.deleted_at` check

The orders endpoint was a red herring — it doesn't join or expose user data. The leak was entirely in the user service's stale cache.
