Here's what was happening:

- **`getProduct()`** queries the DB directly — always fresh. The PATCH endpoint uses this, so the response correctly shows the new price.
- **`getProductPrice()`** uses an in-memory `priceCache` object that's populated on first access and **never invalidated**. Order creation calls this function for price validation, so it keeps seeing the old cached price.

**The fix:** Added `invalidateProductPriceCache()` which deletes the cached entry for the updated product, and call it in the PATCH endpoint right after the DB update. The next call to `getProductPrice()` will re-query the DB and cache the fresh price.
