Done. Here's what changed:

**`src/middleware/auth.js`** — After normal JWT auth resolves, checks for `?as_user=<id>`:
- Non-admins get `403 Forbidden`
- Invalid user ID gets `404 Not Found`
- On success: swaps `req.user` to the target user, stashes original admin as `req.adminUser`
- Middleware is now `async` to support the DB lookup

**`src/middleware/logger.js`** — Logs impersonation context:
- Structured `impersonation: { adminId, actingAs }` in the log entry (available to error tracking)
- Console output shows `[admin:5 as user:123]` tag when impersonating

Usage: `GET /api/orders?as_user=123` with an admin JWT → request executes as user 123. Works on all authenticated endpoints automatically since it's in the global auth middleware.
