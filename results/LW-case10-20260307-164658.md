Done. The new `PATCH /api/orders/:id/status` endpoint:

- **Validates the status** against `ORDER_STATUSES` (`pending`, `confirmed`, `shipped`, `delivered`, `cancelled`) — rejects arbitrary values
- **Blocks setting status to `'deleted'`** — that value isn't in `ORDER_STATUSES`, so it can only be set via `softDeleteOrder`
- **Prevents updating deleted orders** — returns 404 if the order's current status is `'deleted'`

Request body: `{ "status": "confirmed" }`
