The bug was in `src/services/order-service.js:25-27` — a backend issue, not frontend. The count query was building a **new** query against `db('orders')` that only excluded deleted orders, instead of reusing the already-filtered `query` that includes the status filter. So when a user filtered by e.g. "pending", the data came back correctly (3 pending orders), but the total count reflected all non-deleted orders (47).

The fix clones the filtered `query` (which already has both the `!= 'deleted'` and the status filter applied) and uses that for the count.
