**`PATCH /api/orders/:id`** — updates an order before it ships.

Key behaviors:
- **Allowed fields**: `product_name`, `product_id`, `quantity`, `total_cents`, `status`
- **Pre-shipment guard**: Only orders with status `pending` or `confirmed` can be modified. Orders that are `shipped`, `delivered`, or `cancelled` return a `409 Conflict`.
- **Validation**: Checks for valid ID, valid status values, positive integers for `quantity` and `total_cents`, and at least one valid field in the request body.
- **Response**: Returns the full updated order (same shape as `GET /api/orders/:id`).
