Done. Here's what was added:

**New file:** `src/middleware/impersonate.js` — global middleware that intercepts `?as_user=<id>` on any endpoint.

**Behavior:**
- No `?as_user` → no-op, zero overhead
- `?as_user=123` with `role: 'admin'` in JWT → swaps `req.user` to that user, stashes admin in `req.adminUser`
- `?as_user=123` without admin role → `403`
- Invalid/nonexistent user ID → `400` / `404`

**Audit trail:** Logger now includes `[admin:5 as user:123]` in console output and stores `impersonation` in error log entries.

**Middleware order:** `authenticate` → `impersonate` → `rateLimiter` → routes. Impersonation happens before rate limiting, so the admin's own rate limit applies (not the impersonated user's).
