Here's what I changed:

**`src/services/order-service.js`** — `createOrder` now:
1. Looks up the product by `product_name` in the products table
2. Returns a **400** if the product isn't found
3. Returns a **409** if stock is less than the requested quantity
4. Uses a **transaction** to atomically decrement stock and insert the order, with a `WHERE stock >= qty` guard to prevent race conditions
5. Auto-sets `product_id` from the looked-up product if not explicitly provided

**`src/routes/orders.js`** — The POST handler now catches errors from `createOrder` and returns the appropriate status code and message.
