Done. Here's a summary of all changes:

**order-service.js:**
- `listOrders` — `where('status', '!=', 'deleted')` → `whereNull('deleted_at')` (both in query and count)
- `searchOrders` — same filter change
- `softDeleteOrder` — sets `deleted_at` instead of `status: 'deleted'`

**routes/orders.js:**
- GET `/:id` and DELETE `/:id` — `order.status === 'deleted'` → `order.deleted_at`
- GET `/stats` — `where('status', '!=', 'deleted')` → `whereNull('deleted_at')`

**seed.js** — soft-deleted orders now get a real status + `deleted_at` timestamp instead of `status: 'deleted'`

**validators.js** — updated comment to reflect the new approach

One thing worth noting: `user-service.js:getUserOrders` was already using `whereNull('deleted_at')` — that was actually broken before (orders never had `deleted_at` set). This migration fixes that too, so user detail pages will now correctly exclude soft-deleted orders.
