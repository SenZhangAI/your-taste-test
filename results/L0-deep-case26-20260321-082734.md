**The bug:** `priceCache` in `src/services/product-service.js:7` is a simple in-memory object that caches product prices on first read but is never cleared. When the PATCH endpoint updates a price in the DB, `getProductPrice()` keeps returning the stale cached value.

**The fix:** Added `invalidatePriceCache()` that deletes the cache entry for a product, and call it in the PATCH handler right after the DB update — so the next call to `getProductPrice()` will re-fetch the current price from the database.
