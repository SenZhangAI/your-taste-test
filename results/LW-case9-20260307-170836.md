Here's a summary of the 4 memory leaks found and fixed:

| # | File | Issue | Fix |
|---|------|-------|-----|
| 1 | `src/middleware/rate-limiter.js` | `requestLog` array grows on every request, never trimmed | Capped to last 1000 entries |
| 2 | `src/middleware/logger.js` | `recentErrors` array grows on every error response, never trimmed | Capped to last 50–100 entries |
| 3 | `src/services/user-service.js` | `userCache` Map grows per unique user, never expires | Added 10-minute TTL; stale entries replaced on next access |
| 4 | `src/services/product-service.js` | `priceCache` object grows per unique product, never expires | Converted to Map with 5-minute TTL |

The highest-impact leak is #1 (`requestLog`) since it grows on **every single request**. That alone would explain the steady memory climb you're seeing. The caches (#3, #4) are bounded by entity count but still problematic since they hold stale data indefinitely and prevent GC of the cached objects.
