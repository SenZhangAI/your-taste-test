Here's a summary of all changes:

**Orders — `deleted_at` as source of truth:**

1. **`order-service.js:softDeleteOrder`** — now sets `deleted_at` alongside `status: 'deleted'`
2. **`order-service.js:listOrders`** — filters by `whereNull('deleted_at')` instead of `status != 'deleted'`
3. **`order-service.js:getOrder`** — added `whereNull('deleted_at')` filter (was returning deleted orders)
4. **`routes/orders.js`** — removed redundant `order.status === 'deleted'` checks in GET and DELETE routes, since `getOrder` now handles it

This also fixes a real bug: `user-service.js:getUserOrders` was already filtering by `whereNull('deleted_at')`, but `deleted_at` was never being set — so soft-deleted orders were still appearing in user order lists.

**Products — schema mismatch fix:**

5. **`product-service.js:listProducts`** — `is_active` (doesn't exist) → `status: 'active'` (matches the schema)
6. **`routes/products.js`** — `!product.is_active` → `product.status !== 'active'`

The products table has a `status` column, not `is_active` — those queries were hitting a non-existent column.
