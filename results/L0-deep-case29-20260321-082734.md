Done. The impersonation logic is added to the auth middleware (`src/middleware/auth.js:35-47`), so it applies to all endpoints automatically.

**How it works:**
- Any request with `?as_user=123` will have `req.user.id` overridden to `123`
- Only users with `role: 'admin'` can use it — others get a `403`
- Invalid values (non-positive integers) return a `400`
- The impersonated user's role is set to `'user'` to prevent privilege escalation
- The original admin identity is preserved in `req.originalUser` and `req.user.impersonatedBy` for audit/logging purposes
